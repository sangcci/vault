# Question: 디자인 패턴을 생성, 구조, 행위 패턴으로 나눈 기준은 무엇일까?

## 🧐 질문의 배경
GoF(Gang of Four) 디자인 패턴은 크게 세 가지 범주로 나뉩니다. 단순히 외우는 것이 아니라, 이 분류가 어떤 '의도(Intent)'의 차이에서 기인했는지 이해하고 싶습니다.

## 💡 탐색 내용
1.  **생성(Creational) 패턴**:
    - **기준**: "객체 생성 과정을 어떻게 추상화할 것인가?"
    - **핵심**: 객체를 생성할 때 `new`를 직접 쓰지 않고 인터페이스를 통해 생성함으로써 시스템의 유연성을 높입니다.
2.  **구조(Structural) 패턴**:
    - **기준**: "클래스나 객체들을 어떻게 조합해 더 큰 구조를 만들 것인가?"
    - **핵심**: 상속이나 합성을 이용해 인터페이스를 맞추거나 객체들을 묶어 새로운 기능을 만듭니다.
3.  **행위(Behavioral) 패턴**:
    - **기준**: "객체 간의 상호작용과 책임 분산을 어떻게 할 것인가?"
    - **핵심**: 한 객체가 수행하기 어려운 작업을 여러 객체로 분산하거나, 객체 간의 통신 알고리즘을 정의합니다.

## 🚀 추가 질문
- 실무에서 가장 빈번하게 사용되는 패턴은 무엇이며, 그 이유는 해당 분류의 특징과 관련이 있을까?
- 싱글톤(생성) vs 프록시(구조) vs 전략(행위) 패턴의 실제 사용 환경 비교.
