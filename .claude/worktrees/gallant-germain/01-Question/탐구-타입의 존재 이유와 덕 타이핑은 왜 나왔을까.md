# Question: 타입의 존재 이유와 덕 타이핑은 왜 나왔을까?

## 🧐 질문의 배경
정적 타입 언어(Java)와 동적 타입 언어(Python, JS)의 차이를 넘어서, 왜 프로그래밍 세계에는 '타입'이라는 제약이 존재하는지, 그리고 덕 타이핑(Duck Typing)은 어떤 자유를 주는지 궁금합니다.

## 💡 탐색 내용
1.  **타입의 존재 이유**:
    - **컴퓨터 관점**: 메모리 공간의 크기와 해석 방법을 결정합니다. (4바이트를 정수로 읽을지 문자로 읽을지)
    - **인간 관점**: 의도를 명확히 하고, 컴파일 시점에 실수를 방지(Safety)합니다.
2.  **덕 타이핑 (Duck Typing)**:
    - "오리처럼 걷고 오리처럼 꽥꽥거리면 그것은 오리다."
    - 상속이나 인터페이스 구현 여부와 상관없이, **특정 메서드나 속성을 가지고 있다면** 해당 타입으로 간주하는 방식입니다.
    - **등장 배경**: 정적 타입의 경직성을 탈피하고, 다형성을 극도로 유연하게 활용하기 위해 등장했습니다. (Python 등에서 강조)
