# 개념: 가상 메모리와 mmap의 관계

## 📝 정의
`mmap()`은 운영체제의 **가상 메모리 시스템**을 활용하여 디스크의 파일을 프로세스의 주소 공간에 직접 매핑하는 인터페이스입니다.

## ⚙️ 계층적 구조

| 계층 | 요소 | 역할 | 비유 |
| :--- | :--- | :--- | :--- |
| **Interface** | **mmap()** | 프로세스에게 가상 주소를 부여하고 파일과 연결함. | **부동산 계약서** |
| **Mechanism** | **페이징 (Paging)** | 가상 주소를 물리 주소로 변환하고 4KB 단위로 관리함. | **지적도/번지수** |
| **Logistics** | **스와핑/수요 페이징** | RAM에 없는 데이터를 디스크에서 실어나름. | **물류 운송 시스템** |

## 🔄 동작 원리
1. **매핑**: `mmap()` 호출 시 실제 데이터를 읽지 않고 가상 주소 공간에 영역만 확보함.
2. **접근**: 프로세스가 해당 주소에 접근 시 데이터가 RAM에 없으면 **Page Fault** 발생.
3. **로딩**: OS 하부의 **수요 페이징(Demand Paging)** 시스템이 파일에서 4KB 조각을 RAM으로 복사함.
4. **반복**: RAM이 부족해지면 안 쓰는 페이지를 다시 디스크로 내보냄(**Swapping**).

## 💡 본질과의 연결
- **`[[본질-추상화 (Abstraction)]]`**: 개발자는 복잡한 파일 I/O 시스템 콜(`read`, `seek` 등)을 쓰지 않고도, 파일을 마치 거대한 배열처럼 다룰 수 있는 추상화된 인터페이스를 얻게 됩니다.
- **성능**: `mmap()`은 유저 공간과 커널 공간 사이의 데이터 복사를 줄여주지만, OS의 통제 하에 있으므로 세밀한 제어가 어렵습니다.
