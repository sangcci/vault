## 한 문장 정의 (Definition)

- 광신호를 전기신호로 변환하는 망 종단 장치(ONT)와 여러 기기에 IP를 분배하고 보안 기능을 제공하는 공유기(Router)의 역할 및 계층적 차이.

## 네트워크 경로 (Network Path)

```text
[위험: 벽면 직결]
Internet ----> ONT ----> [Home Server (Public IP)]
                          (모든 포트 노출, 서버가 직접 방어해야 함)

[권장: 공유기 경유]
Internet ----> ONT ----> [Router (Firewall/NAT)] ----> [Home Server (Private IP)]
                                  |
                          (포트포워딩된 것만 허용)
```

## 정의 (Detail)

- **ONT (Optical Network Terminal)**: 외부 광케이블을 통해 들어온 빛 신호를 디지털 전기 신호로 변환해주는 장치 (망 종단점).
- **공유기 (Router)**: 하나의 공인 IP를 내부의 여러 기기가 나눠 쓸 수 있게 하고(NAT), 보안 기능을 제공하는 장치.

## 구조적 차이점 (Key Differences)

| 항목 | ONT 직접 연결 | 공유기 경유 연결 |
| :--- | :--- | :--- |
| **IP 할당** | 통신사 공인 IP 직접 할당 | 사설 IP 할당 (192.168.x.x) |
| **보안(방화벽)** | 없음 (PC/서버가 노출됨) | 하드웨어 방화벽 기능 제공 |
| **NAT 기능** | 없음 | 지원 (IP 공유 가능) |
| **포트 제어** | 모든 포트 개방 상태 | 포트 포워딩 필요 |

## 홈서버 구축 시의 시사점 (Implications)

- **벽면 포트 직결의 위험**: 벽면 포트가 공유기를 거치지 않고 ONT와 직접 연결된 경우, 서버가 공인 IP를 받게 되어 외부 공격에 무방비로 노출됨.
- **방화벽 부재**: 공유기가 제공하는 기본적인 보안 계층이 사라지므로, 리눅스 서버 내부에서 `ufw`, `iptables`, `firewalld` 등을 통해 엄격한 인바운드 규칙을 직접 설정해야 함.
- **공유기 기능 활용**: 서버를 공유기 하단에 두어야 NAT를 통한 내부망 구성과 DHCP 관리, 포트 포워딩을 통한 선별적 외부 노출이 가능함.
