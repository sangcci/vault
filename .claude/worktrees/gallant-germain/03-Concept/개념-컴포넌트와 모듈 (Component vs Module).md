# Concept: 컴포넌트와 모듈 (Component vs Module)

## 🎯 한 문장 정의
> (사전적) 모듈은 프로그램의 논리적인 구성 단위이며, 컴포넌트는 독립적으로 배포 및 실행될 수 있는 물리적인 부품 단위.
> (이해용) **"모듈은 설계도에 따라 나뉜 '방'의 구분이고, 컴포넌트는 언제든 갈아 끼울 수 있는 '가전제품'과 같다."**

## 🏗️ 해결하는 문제
- **복잡성 관리**: 시스템을 작은 단위로 나누어 이해하기 쉽게 만듭니다.
- **재사용성**: 잘 정의된 단위는 다른 프로젝트나 시스템에서도 다시 사용할 수 있습니다.
- **독립적 진화**: 특정 부분만 수정하거나 교체해도 전체 시스템에 미치는 영향을 최소화합니다.

## 🔍 주요 차이점
| 구분 | 모듈 (Module) | 컴포넌트 (Component) |
| :--- | :--- | :--- |
| **관점** | 구현 관점 (Logical) | 배포/실행 관점 (Physical) |
| **단위** | 코드, 함수, 클래스의 집합 | 빌드된 바이너리, 라이브러리 (jar, dll) |
| **결합 시점** | 컴파일 타임 (Compile-time) | 런타임 (Run-time) |
| **특징** | 내부 응집도를 높이기 위한 분할 | 인터페이스를 통한 독립적 교체 가능성 |

## ⚖️ 치르는 비용
- **관리 오버헤드**: 단위가 너무 작아지면(Over-modularization) 오히려 전체 구조를 파악하기 힘들고 통신 비용이 증가합니다.
- **버전 관리**: 컴포넌트 간의 의존성 및 버전 불일치 문제가 발생할 수 있습니다.
