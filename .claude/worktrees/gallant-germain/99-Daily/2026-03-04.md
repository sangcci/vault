# 기록

### application properties 적용 규칙들
application.yml이 먼저 적용 -> 그 뒤에 application-{profile}.yml이 덮어쓰임. 아예 대체되는것은 아님.

#### main값과 test값의 충돌 가능성
설정 값들을 덮어씌우는 구조라서, main에 적용된 설정 중에 차마 몰랐던 설정을 모르고 test 환경에서 그대로 실행 -> 설정값이 꼬여서 안되는 경우가 있을듯. 디버깅 추적이 힘들 것으로 예상.

하지만 test는 main위주로 돌아가야 된다고 생각. 그 중에서 테스트하기 어려운 것들만 test container나 다른 외부 격리된 전용 api를 사용하는 것이 현명. 그래서 test 설정도 test 전용이 아닌 main이라고 생각하고 설계해야 할듯.

환경 변수로 따로 두어서 관리하는 것도 현명. 하지만 main에만 있고 test에는 없는 설정 자체를 주입하면 안될듯. 이런 상황이 생기면 안됨.
