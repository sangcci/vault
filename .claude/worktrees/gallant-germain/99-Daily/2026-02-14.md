# 기록
#### Unix Pipe
Goodnote에 설계 철학 - 10-프로세스 통신 노트에 마인드맵 정리함. 그 후 문서들 정리함.
- [[본질-역압 (Backpressure)]] - 공급과 소비의 속도 조절 원리
- [[개념-차단 IO (Blocking IO)]] - 제어권 반납과 대기 메커니즘
- [[개념-프로세스 스케줄러 (Process Scheduler)]] - OS의 프로세스 상태 관리 주체
- [[개념-유닉스 파이프 (Unix Pipe)]] - 커널 버퍼 기반 IPC와 쉘의 연결 사례
- [[판단기준-자원 생명주기 관리와 명시적 종료 (Cleanup)]] - 자원 해제 및 EOF 전달의 중요성
- [[탐구-파이프는 어떻게 무한한 데이터를 유한한 메모리로 처리하는가]] - 파이프의 효율적 데이터 처리 원리 탐구

#### DB Storage & OS Interaction
- [[본질-최소 단위의 불일치 (Granularity Mismatch)]] - 레이어 간 관리 단위 차이로 인한 정합성 붕괴 원리
- [[개념-가상 메모리와 mmap의 관계]] - OS의 메모리 관리 전략과 API 계층 구조
- [[개념-DBMS의 역할과 저장소 관리자 (Storage Manager)]] - DB 내부의 데이터 물류 및 영속성 관리 주체
- [[개념-DB 페이지와 Slotted Page 구조]] - 물리적 I/O 단위와 페이지 내부 레코드 관리 기법
- [[개념-페이지와 인덱스의 추상화 관계]] - 논리적 검색 구조와 물리적 컨테이너의 계층적 연결
- [[탐구-DBMS는 왜 OS의 가상 메모리를 믿지 않을까]] - DB 엔진이 OS의 편리한 기능을 거부하는 기술적 근거
- [[판단기준-시스템 제어권 회수와 자원 독점]] - OS의 범용적 관리를 넘어 직접 제어권을 가져오는 의사결정 기준# 회고

#### 카네기 멜론 데이터베이스 강의
[CMU 15-445/645 Database Systems Fall 2025](https://15445.courses.cs.cmu.edu/fall2025/schedule.html)

데이터베이스의 논리적 부분이 아닌 싫제 물리적 부분을 다룸. 로그, 회복, 버퍼 풀 세팅 등 다양한 물리적인 것들을 실제로 분석하고 튜닝할 수 있게 함. 하루 1강의 정도는 무조건 들어야 될 듯 하다. 이것이 앞으로의 백엔드로써 방향성이 될 것 같다.
