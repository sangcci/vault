**핵심 질문**: "어떻게 물리적 실체를 바꾸지 않고도 대상마다 서로 다른 현실을 보여줄 수 있는가?"

## 한 문장 정의 (Definition)

> (사전적) 요청(Request)과 자원(Resource) 사이에 중간 층(Indirection Layer)을 두어, 직접적인 연결 대신 매핑(Mapping)을 통해 대상을 참조하는 원리.
> (이해용) 프로세스가 '진짜 세상'을 직접 보지 못하게 중간에 '커널이라는 중재자'를 두고, 중재자가 프로세스의 안경(Context)에 따라 서로 다른 현실을 그려주는 기법.

## 사용 예시 (Examples)

1. **OS (가상 메모리)**: 프로세스가 보는 주소(가상 주소)와 실제 RAM의 주소(물리 주소) 사이에 페이지 테이블이라는 간접 층을 두어 메모리 격리 구현.
2. **Network (DNS)**: 사용자가 입력하는 이름과 실제 서버의 IP 사이에 DNS라는 간접 층을 두어 유연한 주소 관리 가능.
3. **Docker (Namespaces)**: 프로세스의 시스템 콜 요청과 실제 커널 자원 사이에 네임스페이스라는 간접 층을 두어 프로세스별로 서로 다른 PID, 네트워크 환경을 제공.
4. **OS (init → systemd)**: 커널은 `/sbin/init`이라는 이름만 탐색. 실제 파일은 `systemd`를 가리키는 심볼릭 링크. 커널은 무엇이 실행되는지 알지 못하고 관심도 없음 → 구현체 교체가 자유로움.

## 트레이드오프 (Trade-off)

- **장점 (격리와 유연성)**: 실체를 바꾸지 않고도 논리적으로 완벽한 격리를 구현할 수 있음. "서로 내부를 모른다"는 상태를 만들어 보안과 안정성을 확보.
- **단점 (오버헤드)**: 직접 참조하는 것보다 중간 층을 한 번 더 거쳐야 하므로(매핑 연산, 컨텍스트 스위칭 등) 미세한 성능 손실이 발생함.

## 왜 사라지지 않는가 (Persistence)

컴퓨터 과학의 격언인 **"모든 문제는 또 다른 간접 층을 도입함으로써 해결할 수 있다"**는 말처럼, 간접 참조는 복잡한 시스템에서 '격리'와 '추상화'를 실현하는 유일무이한 방법이기 때문입니다. 하드웨어가 아무리 발전해도 소프트웨어 간의 충돌을 막기 위한 이 원리는 변하지 않습니다.

## 다른 모습들 (Polymorphism)

### "서로 내부를 모른다" (Mutual Unawareness)
이 표현은 간접 참조 원칙에서 기인합니다. 프로세스는 중간 층이 보여주는 '필터링된 결과'만 믿기 때문에, 자신과 똑같은 물리 자원을 공유하는 다른 존재가 있는지조차 인지할 수 없습니다.

### ASCII Diagram: Indirection Layer in Isolation
```text
[ Process A (User Mode) ]      [ Process B (User Mode) ]
          |                              |
          | (Request: "Who am I?")       | (Request: "Who am I?")
          V                              V
+------------------------------------------------------+
|             Kernel (Indirection Layer)               |
|  - Check Namespace A tag   | - Check Namespace B tag  |
|  - Return "PID 1"          | - Return "PID 1"         |
+------------------------------------------------------+
          |                              |
          V                              V
[ Actual Physical Resource (Global PID: 48372, 48373)  ]
```


> 사람들의 지능을 나누는 한 가지 방법은 동시에 얼마나 많은 서로 다른 것을 마음속에 담아두고 그 아이디어를 연결할 수 있는지 알아보는 것입니다. 제가 기억하기로는, 동시에 7개의 "것"을 마음속에 담아둘 수 있다면 이미 엄청 똑똑한 겁니다.
> 하지만 간접 참조를 사용하면, 그것들을 그룹화하여 생각하는 것들의 수를 줄일 수 있고, 더 높은 수준의 이해로 작업할 수 있게 됩니다.
