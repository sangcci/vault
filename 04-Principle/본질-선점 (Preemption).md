# Principle: 선점 (Preemption)
**핵심 질문**: "누가 CPU라는 한정된 자원의 통제권을 쥐고 있는가? (사용자인가, 운영체제인가?)"

## One-liner Definition
> (사전적) 운영체제(커널)가 실행 중인 프로세스로부터 강제로 CPU 제어권을 빼앗아 다른 프로세스에게 할당할 수 있는 방식.
> (이해용) **"심판(OS)이 호루라기를 불어 경기를 중단시키고, 선수(프로세스)를 교체하는 강력한 권한."**

## Usage Examples (문장 3개)
1. "시분할 시스템은 **선점** 스케줄링을 통해 여러 프로그램이 동시에 실행되는 것처럼 보이게 한다."
2. "우선순위가 높은 프로세스가 CPU를 **선점**하면서 실시간 응답성을 확보했다."
3. "**선점**으로 인한 문맥 교환 오버헤드가 커지면 시스템 전체의 처리량이 감소할 수 있다."

## Recurring Core Problem
컴퓨터의 자원(CPU)은 한정되어 있는데, 하나의 프로세스가 자원을 독점하면 다른 모든 프로세스가 굶주리는(Starvation) 현상이 발생합니다. 특히 프로그램이 무한 루프에 빠졌을 때 시스템 전체가 마비되는 것을 어떻게 막을 것인가가 핵심 문제입니다.

## Why It Doesn't Disappear
현대 컴퓨팅의 핵심인 '멀티태스킹'과 '반응성'을 보장하기 위해서는 강제적인 자원 배분이 필수적이기 때문입니다. 하드웨어 타이머와 커널의 개입이 없다면 사용자 경험(UX)은 극도로 저하될 수밖에 없습니다.

## Trade-offs
- **공정성 vs 오버헤드**: 모든 프로세스에 기회를 골고루 주지만, 작업을 멈추고 상태를 저장/복구하는 '문맥 교환(Context Switch)' 비용이 발생합니다.
- **응답성 vs 경쟁 조건**: 즉각적인 작업 전환이 가능해지지만, 공유 자원을 사용하던 도중 작업이 중단되면 데이터가 꼬이는 '경쟁 조건' 위험이 커집니다.

## Appears As
- **Round Robin**: 정해진 시간만큼 번갈아 실행
- **Interrupt**: 하드웨어 신호에 의한 강제 작업 전환
- **Time Slice/Quantum**: 프로세스에 할당된 최소 시간 단위
