# Principle: 일관성의 종류 (Consistency Types)
**핵심 질문**: "데이터가 어디에 있든, 언제 읽든 서로 모순되지 않도록 지켜주는 약속은 무엇인가?"

## One-liner Definition
> (사전적) 어떤 사실이나 현상의 전후가 모순 없이 한결같은 상태.
> (이해용) **"데이터가 어디에 있든, 언제 읽든 서로 모순되지 않도록 지켜주는 약속"**. (ACID, CAP, Model 모두 본질은 같으나 적용되는 **범위(Scope)**가 다를 뿐입니다.)

## Usage Examples (문장 3개)
1. "분산 시스템에서 선형화(Linearizability) 모델을 선택하여 모든 노드가 즉각적으로 동일한 최신값을 보도록 **일관성**을 강화했다."
2. "글로벌 서비스의 가용성을 높이기 위해 최종 **일관성**(Eventual Consistency) 모델을 채택하고 데이터 수렴 속도를 최적화했다."
3. "ACID 트랜잭션의 **일관성**은 데이터베이스 내의 무결성 제약조건이 깨지지 않음을 보장한다."

## Recurring Core Problem
데이터가 여러 곳에 복제되거나 여러 단계에 걸쳐 처리될 때, 하드웨어 장애나 네트워크 지연으로 인해 정보가 서로 달라지는 현상이 발생합니다. "누가 어떤 데이터를 언제 보느냐"에 따라 결과가 달라지는 모순을 어떻게 관리하고 합의할 것인가가 핵심 문제입니다.

## Why It Doesn't Disappear
현대 시스템은 성능과 확장성을 위해 데이터를 분산 저장하고 병렬 처리하는 것이 필수적이기 때문입니다. 분산 환경에서는 '완벽한 실시간 일치'와 '빠른 응답 속도' 사이에서 반드시 하나를 선택해야 하는 물리적 제약(CAP 정리)이 존재하므로, 비즈니스 상황에 맞는 일관성 선택은 영원한 과제입니다.

## Trade-offs
- **강한 일관성 vs 가용성**: 모든 노드의 일치를 기다리면 응답이 느려지거나 일부 노드 장애 시 시스템이 멈춥니다(CP).
- **약한 일관성 vs 데이터 정확성**: 빠른 응답을 위해 일치를 미루면(AP), 사용자가 잠시 동안 이전 데이터를 보거나 데이터 충돌이 발생할 수 있는 위험을 감수해야 합니다.

## Appears As
- **Linearizability**: 가장 강력한 즉각적 일치
- **Eventual Consistency**: 결국에는 같아지는 느슨한 일치
- **ACID-C**: 단일 DB의 무결성 규칙 준수

## 🔭 관점의 차이: 일관성의 범위 (Scope)
용어가 혼란스러운 이유는 일관성의 **본질**이 아니라, 그 약속이 적용되는 **범위**가 다르기 때문입니다.

1.  **ACID-C (데이터 무결성):** 범위가 **'단일 데이터베이스'** 내부의 데이터 규칙(PK, FK)입니다.
2.  **CAP-C (데이터 동일성):** 범위가 **'분산된 여러 노'** 간의 데이터 일치 여부입니다.
3.  **Consistency Models (순서/가시성):** 범위가 **'시간의 흐름에 따른 읽기/쓰기 연산의 순서'**입니다.

## 📊 계층 구조로 이해하기 (Layered View)

일관성은 추상적인 목표부터 실제 구현까지 3개의 레이어로 나뉩니다.

| Layer | 구분 | 한 줄 정의 | 핵심 속성 |
| :--- | :--- | :--- | :--- |
| **Layer 1** | **CAP Theorem** | 네트워크 분할 시 선택할 **전략적 목표** | 이진법 (Yes/No) |
| **Layer 2** | **Consistency Models** | 가시성과 순서를 정의하는 **구체적 명세** | 스펙트럼 (강함 → 약함) |
| **Layer 3** | **Implementation** | 모델을 실제로 구현한 **기술 스택** | 알고리즘 (Raft, Gossip 등) |

## 🔄 분산 일관성 모델 스펙트럼 (Layer 2)

### 1️⃣ Strong Consistency (CAP-C 만족 ✅)
*   **Linearizability (선형화):** 가장 강력함. 쓰기 완료 즉시 모든 노드에서 보임 (실시간 순서 보장).
*   **Sequential Consistency (순차):** 모든 노드가 동일한 순서로 사건을 봄 (실시간 보장은 없음).

### 2️⃣ Weak Consistency (CAP-C 희생 ❌)
*   **Causal Consistency (인과):** 원인과 결과 관계가 있는 데이터만 순서 보장 (채팅, 댓글).
*   **Eventual Consistency (최종):** 당장은 달라도 결국 같아짐 (SNS 좋아요, 캐시).
