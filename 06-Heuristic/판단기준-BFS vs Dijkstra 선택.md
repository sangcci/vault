## 판단 기준

> **간선 가중치가 균등한가?**
> - 균등 (모두 동일) → BFS
> - 불균등 (제각각) → Dijkstra

```
가중치 균등?
    │
   YES → BFS  O(V + E)        큐(FIFO), 정렬 불필요
    │
    NO → Dijkstra  O(E log V) 힙(우선순위 큐), 매번 최솟값 추출
```

## 왜 가중치가 기준인가

```
균등 가중치:
  먼저 꺼낸 노드 = 항상 최단 거리
  → 순서 변경 불필요 → FIFO 큐로 충분

불균등 가중치:
  나중에 발견된 노드가 더 짧을 수 있음
  → 항상 현재 최솟값을 먼저 처리해야 함
  → 힙 필요 → O(log N) 발생
```

## 효과적인 상황

**BFS**
- 이동 횟수 최소화 (BOJ 숨바꼭질, 미로 탈출)
- 상태 공간 탐색 (암묵적 그래프)
- 간선 가중치가 1 또는 동일한 값

**Dijkstra**
- 도로 네트워크 (거리/시간이 제각각)
- 비용 최소화 경로

## 실패하는 상황

**BFS를 불균등 가중치에 쓰면**: 먼저 도착한 경로가 최단이라는 보장 없음 → 오답

**Dijkstra를 균등 가중치에 쓰면**: 정답은 나오지만 O(E log V)로 불필요하게 느림
