## 판단 기준

- **"아직 방문 안 한 정점이 있는가?"가 핵심이면** → 정점 중심 (1~n 순회 + visited 배열)
- **간선 자체가 처리 대상이면** → 간선 중심 (간선 리스트 순회)

## 효과적인 상황

**정점 중심이 필요한 경우:**
- 연결 요소 개수 계산 (BOJ 11724)
- 도달 가능한 정점 집합 탐색
- 고립 정점(차수 = 0)이 존재할 수 있는 문제

```
// 정점 중심: 1~n을 외부 루프로 순회
for v in 1..=n:
    if not visited[v]:
        component_count += 1
        BFS/DFS(v)          // 이 시작점이 새 컴포넌트 진입점
```

**간선 중심이 필요한 경우:**
- 최소 신장 트리 (Kruskal — 간선을 가중치 순 정렬 후 처리)
- 간선에 조건/가중치가 있고 간선 자체를 선택/거부하는 문제

```
// 간선 중심: 간선 리스트를 외부 루프로 순회
for (u, v, w) in sorted_edges:
    if find(u) != find(v):
        union(u, v)
```

## 실패하는 상황

**간선 중심으로 연결 요소를 세려 할 때:**

```
// 간선 목록: (1,2), (3,4)
// 정점 5는 고립 → 간선이 없어 순회 자체가 불가
edges = [(1,2), (3,4)]
for (u, v) in edges:  // 정점 5에 접근할 방법이 없음
    ...
// 정답: 3 (컴포넌트: {1,2}, {3,4}, {5})
// 간선 중심 결과: 2 → 오답
```

핵심 실패 원인: **간선 순회는 차수가 0인 정점을 구조적으로 탐지할 수 없다.** 시작점을 어디서 잡을지 알려면 정점 전체를 외부에서 한 번씩 확인해야 한다.

## 선택 기준 요약

```
문제가 "정점을 모두 처리했는가?"를 묻는가?
    YES → 정점 중심 (visited[1..n] 외부 루프)
    NO  → 간선이 처리 단위인가?
              YES → 간선 중심 (edges 리스트 순회)
```
