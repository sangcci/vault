## 판단 기준

```
최적화 문제인가?
       ↓ Yes
그리디 선택 속성 성립? (지역 최적 = 전역 최적)
   ↓ Yes                    ↓ No
그리디 사용            최적 부분 구조 있는가?
                         ↓ Yes       ↓ No
                        DP 사용    Brute Force
```

- 선택 후 **이전 상태를 잊어도 된다** → 그리디
- 선택이 **미래 선택에 영향을 주고 기억이 필요하다** → DP

## 효과적인 상황

| 알고리즘 | 적합한 문제 유형 |
|---------|----------------|
| 그리디 | Interval Scheduling, Huffman Coding, Dijkstra, Kruskal |
| DP | 0/1 배낭, LCS, 행렬 연쇄 곱셈, 음수 가중치 최단 경로 |

## 실패하는 상황

- **그리디 실패 1 — 0/1 배낭 문제**: 단가(가치/무게)가 높은 것부터 선택하면 최적이 아닐 수 있음
- **그리디 실패 2 — 음수 가중치 최단 경로**: 지금 가장 짧은 경로가 나중에 더 짧아질 수 있음
- **DP 과적용**: 그리디 선택 속성이 성립하는데 DP를 쓰면 불필요한 메모리와 시간 낭비
