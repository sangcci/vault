## 판단 기준

```text
롤백 보상 로직을 구현할 수 있는가?
  ├─ No  → DB 먼저 쓰기 (Write-Around / Cache-Aside 권장)
  └─ Yes → Write-Through 고려 가능

캐시 미스 발생 시 어떻게 처리할 것인가?
  → "찾을 수 없으면 넘어가고, 이후 요청에서 다시 저장" = Cache-Aside 패턴
```

- **캐시 → DB (Write-Through)**: 캐시 업데이트 성공 후 DB 실패 시 롤백 미구현이면 캐시가 더 최신인 불일치 상태 발생. 별도 보상 로직 없이는 비추.
- **DB → 캐시 (Write-Around)**: DB 성공 확인 후 캐시 갱신. 실패해도 DB가 정답이므로 불일치 없음. **일반적으로 권장.**

---

## 효과적인 상황

- 단순 서비스에서 롤백 보상 로직 없이 운용: DB 먼저 쓰기
- 쓰기 빈도 낮고 읽기 압도적으로 많은 경우: Write-Through (일관성 우선)

---

## 실패하는 상황

- Write-Through에서 DB 실패 롤백 로직 없이 운용 → 캐시-DB 불일치 상태 지속
