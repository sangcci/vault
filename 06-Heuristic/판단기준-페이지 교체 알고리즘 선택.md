## 판단 기준

- 이론적 성능 상한 계산 → **OPT** (미래 참조 알 때만 가능)
- 구현 없이 이해만 필요 → **FIFO** (단, Belady's Anomaly 주의)
- 최근성 기반 캐시 (범용) → **LRU** (Redis, DB 버퍼풀)
- 반복 참조 패턴 캐시 → **LFU** (장기 인기 콘텐츠 CDN)
- OS 커널 실제 구현 → **Clock** (Linux, Unix 계열)

## 알고리즘 비교표

| 알고리즘 | Belady's Anomaly | 구현 복잡도 | 장점 | 단점 |
|---------|-----------------|------------|------|------|
| FIFO    | 발생 가능 ⚠️    | 매우 단순  | 구현 쉬움 | 자주 쓰는 것도 교체 |
| OPT     | 없음             | 구현 불가  | 최소 Fault | 미래 예측 불가 |
| LRU     | 없음             | 중간       | 실용적 최적 | 참조마다 업데이트 비용 |
| LFU     | 없음             | 높음       | 빈도 패턴 적합 | 캐시 오염, 구현 복잡 |
| Clock   | 없음             | 낮음       | LRU 근사, 저비용 | 완전 LRU보다 부정확 |

## 효과적인 상황

- **LRU**: 최근 참조가 미래 참조를 예측하는 일반적인 캐시 (Redis, MySQL)
- **LFU**: 인기 순위가 안정적인 콘텐츠 캐시 (CDN, 미디어 스트리밍)
- **Clock**: OS 커널처럼 수백만 페이지를 저비용으로 관리해야 할 때

## 실패하는 상황

- **LRU**: 순차 스캔 (Sequential Flood) — 새 페이지가 모두 교체됨 (MySQL: MRU로 보완)
- **LFU**: 초기 캐시 웜업 — 새 페이지가 오래된 인기 페이지를 이길 수 없음
- **FIFO**: 자주 참조되는 페이지가 오래됐다는 이유로 교체될 때
