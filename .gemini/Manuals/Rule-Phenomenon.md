# Phenomenon Note Rules (현상)

## Purpose
To document observed **symptoms**, side effects, errors, or anomalies.

## File Naming
- Format: `02-Phenomenon/현상-<Title>.md`
- Example: `현상-Noisy Neighbor.md`

## Template
```markdown
---
aliases: []
tags: [현상, 작성중]
type: Phenomenon
difficulty: Medium
---
# Phenomenon: <Title>

## 한 문장 정의 (Definition)
> (사전적) ...
> (이해용) ...

## 발생 환경 (Context)
- When does this happen?

## 관찰되는 증상 (Symptoms)
- Latency spike? CPU saturation?

## 추측되는 원인 (Root Cause)
- Link to [[개념-...]] or [[본질-...]]

## 관련 사례 (Related Cases)
- [[사례-...]]
```
