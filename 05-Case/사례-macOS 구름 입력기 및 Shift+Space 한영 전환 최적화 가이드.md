# 사례-macOS 구름 입력기 및 Shift+Space 한영 전환 최적화 가이드

## 1. 개요
macOS에서 구름 입력기를 사용하면서 `Shift + Space` 단축키를 통해 한/영 전환을 수행하고, 불필요한 기본 입력 소스를 제거하여 입력 환경을 최적화하는 방법입니다.

## 2. 해결 과정

### 2.1. 구름 입력기 설정 및 단축키 지정
구름 입력기 자체 설정을 통해 `Shift + Space`를 한/영 전환 키로 등록합니다.

1. **단축키 설정 확인/변경**:
    - 화면 상단 구름 입력기 아이콘 클릭 > **환경설정** > **단축키** 탭으로 이동
    - **"한/영 전환"** 단축키 항목이 `Shift + Space`로 되어 있는지 확인합니다.
    - 만약 다르다면 해당 항목을 더블 클릭하여 `Shift + Space`를 눌러 입력합니다.
2. **시스템 단축키 해제 (충돌 방지)**:
    - **시스템 설정** > **키보드** > **키보드 단축키...** > **입력 소스**로 이동
    - **'이전 입력 소스 선택'** (보통 `Command + Space`) 항목을 해제합니다.
    - 이를 통해 macOS 시스템의 전환 동작이 간섭하지 않게 되어, 구름 입력기의 `Shift + Space`가 원활히 작동합니다.

### 2.2. 기본 영어 키보드 강제 삭제 (일본어 Romaji 트릭)
macOS에서 기본 영어 키보드는 일반적으로 삭제가 불가능하도록 잠겨 있습니다. 구름 입력기(한글/쿼티)만 남겨서 전환 오류를 방지하는 방법입니다.

1. **일본어 키보드 추가**: **시스템 설정** > **키보드** > **입력 소스**에서 `+` 버튼을 눌러 **'일본어'**를 추가합니다.
2. **옵션 변경**: 추가된 일본어 키보드의 설정에서 **Input modes** 중 **'Romaji'** 항목만 체크합니다.
3. **기본 영어 키보드 삭제**: 이제 기본 영어 키보드를 선택하고 `-` 버튼을 누르면 삭제가 가능해집니다.
4. **일본어 키보드 삭제**: 기본 영어 키보드를 삭제한 후, 추가했던 일본어 키보드도 삭제합니다.
5. **결과**: 구름 입력기만 남게 되어, `Shift + Space` 시 다른 입력 소스로 튀는 현상 없이 깔끔하게 전환됩니다.

## 3. 결과 및 확인
- **Shift + Space**: 구름 입력기 내에서 한/영이 즉각 전환됩니다.
- **입력 소스 메뉴**: 메뉴 바에 구름 입력기 외에 다른 기본 키보드(ABC 등)가 나타나지 않는지 확인합니다.
