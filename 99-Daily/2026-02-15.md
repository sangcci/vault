# 플랜

### 1. 프로젝트 요구사항 및 문서화 (Notion)
- [x] **Rent/Survey 기능 명세:** Notion에 흩어진 요구사항을 기능 단위로 리스트업 및 계층화
- [x] **비기능 요구사항 정의:** 02-13 기록 기반, 각 프로젝트의 목표 성능(TPS, Latency) 및 가용성 지표 설정
- [x] **설계 산출물 매핑:** ERD, API 명세(Swagger), 인프라 구조도를 Notion 기획 페이지에 연동 및 설명 추가

### 2. 이력서 전략 및 필살기 도출
- [x] **기술적 도전 과제 정리:** '문제 해결 경험' 위주로 Rent/Survey의 핵심 로직(Query 최적화, 데이터 정합성 등) 정리
- [x] **SI 맞춤형 역량 추출:** Mybatis 활용 가능성, 유지보수를 고려한 인터페이스 설계 등 SI에서 선호하는 '기본기' 강조 포인트 도출
- [x] **이력서 초안 스케치:** '무엇을 했다'가 아닌 '어떻게 기여했고 어떤 성과(수치)를 냈는지' 중심의 문구 작성 연습

### 3. 기술 학습 및 CS (CMU DB 강의)
- [x] **CMU 15-445 학습:** 일일 1강의 루틴 준수 (Database Storage 파트 심화)
- [x] **OS/DB 상호작용 복습:** 어제 정리한 'DBMS가 왜 가상 메모리를 신뢰하지 않는가' 등 핵심 탐구 내용 리마인드
- [x] 컨텍스트 스위치 딥 다이브: 하드웨어 수준까지 아예 딥 다이브해서 확실하게 그림으로 이해.

### 4. 루틴 및 복습
- [x] **Anki 복습:** 전체 카드 복습 및 오늘 도출한 비기능 요구사항 수치들을 신규 카드로 등록
- [x] **Red Flag 해결:** 안키 복습 중 이해가 부족했던 개념(Red Flag) 1~2개 집중 파고들기
