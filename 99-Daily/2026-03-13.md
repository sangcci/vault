# 기록

### prometheus-postgres-exporter 설정 (db-tier VM)

**1. PostgreSQL 모니터링 유저 생성**
```sql
sudo -u postgres psql

CREATE USER postgres_exporter WITH PASSWORD 'your_password';
GRANT pg_monitor TO postgres_exporter;
GRANT CONNECT ON DATABASE allreva TO postgres_exporter;
```

**2. pg_hba.conf에 접속 허용 추가**
```
local   all   postgres_exporter   md5
```
```bash
sudo systemctl reload postgresql
```

**3. DSN 환경변수 설정 (Debian 정석 방식)**
```bash
# /etc/default/prometheus-postgres-exporter
DATA_SOURCE_NAME=postgresql://postgres_exporter:password@localhost:5432/allreva?sslmode=disable
```
- `/usr/lib/systemd/system/` 원본 서비스 파일은 수정하지 않음 (패키지 업데이트 시 덮어써짐)
- `systemctl edit`으로 override 생성하는 방법도 있지만, apt 설치 시 `/etc/default/` 파일이 제공되므로 이쪽이 정석

**4. 서비스 시작 및 확인**
```bash
sudo systemctl restart prometheus-postgres-exporter
sudo systemctl status prometheus-postgres-exporter
curl localhost:9187/metrics | head
```

**5. Prometheus scrape 설정**
```yaml
scrape_configs:
  - job_name: 'postgres'
    static_configs:
      - targets: ['<db-tier VM IP>:9187']
```

**참고: PostgreSQL 인증 구조**
- `pg_hba.conf`에서 접속 경로(local/host)별로 인증 방식 결정
- `peer`: OS 유저명 == PG 유저명 확인 (비밀번호 없음)
- `md5`/`scram-sha-256`: PG 자체 비밀번호 인증
- `sudo -u postgres psql`: OS 유저를 postgres로 전환 후 실행 → peer 통과
- `sudo psql -U postgres`: OS 유저가 root → peer 실패

**참고: DSN에 allreva vs postgres**
- 서버 전체 통계(connections, locks)는 어떤 DB든 동일
- 테이블/인덱스별 통계(`pg_stat_user_tables`)는 접속한 DB의 것만 수집
- 모니터링 대상이 allreva이므로 allreva DB로 접속이 맞음

### redis-exporter 설정

공식 Prometheus 등록 exporter: [oliver006/redis_exporter](https://github.com/oliver006/redis_exporter)

Redis가 Docker로 구동 중이면 docker-compose로 같이 구성하는 게 깔끔:
```yaml
redis_exporter:
  image: oliver006/redis_exporter
  environment:
    - REDIS_ADDR=redis://redis:6379
  ports:
    - "9121:9121"
```

기본 포트: **9121**

### node-exporter: Docker 컨테이너 → 호스트 접근

Prometheus가 Docker 안, node-exporter가 호스트에서 구동 중일 때 scrape 설정:

**방법 1: host.docker.internal (가장 간단)**
```yaml
- job_name: 'node'
  static_configs:
    - targets: ['host.docker.internal:9100']
```
Linux에서는 기본 미지원 → `docker run --add-host=host.docker.internal:host-gateway` 추가 필요

**방법 2: docker0 브릿지 IP 직접 지정**
```yaml
- targets: ['172.17.0.1:9100']
```
```bash
# IP 확인
ip addr show docker0
```

**방법 3: host 네트워크 모드**
```bash
docker run --network=host prom/prometheus
```
localhost:9100으로 바로 접근 가능. 단, 컨테이너 네트워크 격리 사라짐

### apt 설치 패키지 제거
```bash
sudo apt purge prometheus-postgres-exporter  # 설정 파일까지 제거
sudo apt autoremove
```
`remove`는 설정 파일 남김, `purge`는 전부 제거
