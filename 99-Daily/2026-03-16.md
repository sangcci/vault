# 기록

### nginx upstream Host 헤더 오류 해결

**문제**: Spring Boot 로그에 `The host [was_backend] is not valid` (Http11Processor)

**원인**: `proxy_pass http://was_backend;` 설정 시 nginx가 upstream 이름을 그대로 Host 헤더에 담아 전송 → Tomcat 8.5+ Host 헤더 strict 검증에서 reject

**해결**: `default.conf`에 `proxy_set_header Host $host;` 추가

```nginx
location / {
    proxy_pass http://was_backend;
    proxy_set_header Host $host;
}
```
