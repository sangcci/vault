### NetworkManager vs networking(ifupdown)
#### networking (/etc/network/interfaces)
- Debian/Ubuntu 전통 방식
- 정적인 환경 에 최적화 — 서버처럼 네트워크가 바뀌지 않는 곳
- ifup, ifdown 명령으로 수동 관리
- 설정 파일을 직접 편집해야 함
- 동적 변화에 약함 (USB 장치 탈착, WiFi 전환 등)
- 데몬 없이 부팅 시 스크립트로 한 번만 실행
#### NetworkManager
- Red Hat이 개발, 현재 대부분 배포판 기본 채택
- 동적인 환경 에 최적화 — 랩탑, 데스크탑처럼 네트워크가 자주 바뀌는 곳
- 백그라운드 데몬으로 항상 실행되며 상태 감지
- nmcli, nmtui, Cockpit GUI로 관리 가능
- WiFi 자동 전환, VPN, 핫스팟 등 고급 기능
- libvirt, Docker와 통합이 잘 됨

왜 둘이 충돌했나
```
/etc/network/interfaces에 enx 설정 존재
          ↓
NetworkManager: "이 장치는 다른 곳에서 관리 중 → 내가 손대지 않을게"
          ↓
enx = unmanaged 상태
          ↓
DHCP 아무도 안 함 → IP 없음
```

### 모니터링 시스템 구축 전 VM 세팅 및 CICD 환경 세팅
VM에 각 시스템이 구축되어 있어야 로그를 집계할 수 있음. 그리고 CICD도 같이 세팅해야 함.

그러면 일단 VM 세팅 및 간단한 조작법 부터 숙지하는게 중요. Host OS를 통해서 어떻게 Guest로 진입하는지를 아는게 중요. 

현재 이슈로는 VM 설치 중 자꾸 `localhost login:`이 발생. 뭔가 잘못 설치되었다는 소리임. 

그래서 cloud 이미지 말고 .iso로 직접 부팅하는 방법으로 변경. 이게 훨씬 좋은듯.
