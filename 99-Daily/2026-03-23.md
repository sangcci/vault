### onedrive 대체
점점 Apple 생태계로 넘어가면서, onedrive를 8900원 내면서 유지할 필요성을 더이상 느끼지 못했다. 특히 office 계열은 잘 쓰지도 않고 최근 onlyoffice라는 아주 좋은 무료 office 대안이 생겼고, icloud 200GB 해도 4,400원 50GB는 1,100원이라는 아주 미친 가격으로 제공한다. onedrive를 유지할 필요가 없어졌다. 심지어 android와도 완벽히 동기화 가능. 사진 데이터는 Google Picture인가 그걸로 무료로 유지 가능.. obsidian도 icloud로 제어 가능. 모바일은 읽기만 하면 동기화 문제도 없음. obsidian sync가 더 좋지만 환율 고려 월 7554원을 낼 자신이 없음. 1100원으로 모든걸 해결 가능함.

- icloud 구독(50GB, 200GB 고민)
- onedrive 데이터를 icloud로 이전.
- microsoft onedrive & 365 office 삭제
- onlyoffice 설치 -> onlyoffice 한글 깨짐 이슈 발생. freeoffice로 대체
- obsidian icloud 교체

### obsidian 노트 마이그레이션
필요한 노트들만 선별해서 이동한다

**database 카테고리 (25개 중 4개 생성)**
- [[개념-N+1 문제 (N+1 Query Problem)]] - 카테시안 곱 비교로 N+1 설명, ORM Lazy Loading 메커니즘, 해결 전략 4가지
- [[개념-JDBC Fetch Size]] - fetch size에 따른 네트워크 왕복 vs 메모리 트레이드오프, 드라이버별 기본값
- [[개념-복합키 (Composite Key)]] - FK 전파 비용, 인덱스 제약, 수강 테이블 예제로 부분 함수적 종속 → 이상 현상 연결
- [[판단기준-SELECT 쿼리 설계]] - JOIN 방식 선택, GROUP BY 인덱스 확인, 시나리오 기반 쿼리 검증

**design 카테고리 (15개 중 1개 생성)**
- [[본질-값의 신뢰성 (Value Trustworthiness)]] - 불변성 + 예측 가능성 + 자기 검증 = VO의 본질, JPA와의 현실적 충돌 포함

**나머지 카테고리 일괄 분석 (devops 14, java 17, spring 21, cs 8, oop 5, 기타 35 = 100개 중 6개 생성)**
- [[개념-Spring 트랜잭션 관리 (Transaction Management)]] - PlatformTransactionManager 추상화, ThreadLocal 커넥션 동기화, @Transactional AOP 기반
- [[개념-AOP (Aspect-Oriented Programming)]] - 횡단 관심사 모듈화, 위빙 전략 3가지, JDK Proxy vs CGLib
- [[개념-Checked vs Unchecked Exception]] - 컴파일 시점 강제 여부, LSP 위반, 트랜잭션 롤백 차이, 커스텀 예외 Unchecked 권장
- [[개념-Tomcat HTTP 요청 처리 흐름]] - TCP→NIO Selector→Worker Thread 할당, 스레드 점유 구간 시각화
- [[본질-이벤트 기반과 비동기의 독립성]] - 이벤트 기반(결합도) ≠ 비동기(성능), ApplicationEventPublisher는 큐가 아님
- [[본질-파일 추상화 (Everything is a File)]] - Unix "모든 것이 파일" 철학, FileDescriptor, Java InputStream 추상화 계층
