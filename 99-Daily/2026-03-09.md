# 기록

## Allreva CI/CD - Self-hosted Runner Blue-Green 무중단 배포

### 배경
- 홈서버 KVM VM에 무중단 배포 구성
- 인바운드 포트 오픈이 어려움 (방화벽, NAT) → Self-hosted Runner 채택 (outbound polling)
- VM 구성: virbr0 브릿지, Web VM (Nginx) + WAS VM (Spring Boot), 둘 다 Docker 기반

### 구성

**GitHub Actions 파이프라인**
```
PR → ci (GitHub hosted): 테스트/spotless
develop push → ci → cd (GitHub hosted): 빌드/DockerHub 푸시
             → deploy-app (self-hosted, was): Blue-Green 배포
             → switch-nginx (self-hosted, web): upstream 전환
```

- `runs-on: [self-hosted, was]` / `[self-hosted, web]` 라벨로 runner 매칭
- APP_NAME은 `allreva-springboot`로 env 하드코딩

**VM 배포 구조**
- App VM: `~/deploy/deploy.sh` - Docker pull → 새 컨테이너 기동 → 헬스체크 → 이전 컨테이너 종료
- Nginx VM: `~/nginx/switch.sh` - upstream 포트 전환 (blue:8081 / green:8082) → nginx reload
- `active-slot` 파일로 현재 활성 슬롯 추적, GitHub Actions outputs로 전달

### Firebase 서비스 계정 리팩토링

**문제**: `ClassPathResource`로 firebase json 경로 하드코딩 → VM 배포 시 경로 변경 불가

**해결**: property화
- `fcm.service-account-key` property 추가
- 기본값: `classpath:firebase/firebase-service-account-key.json`
- prod: `file:/app/firebase/firebase-service-account-key.json` (Docker volume 마운트)
- `FcmInitializer`: `@Value` + `ResourceLoader`로 경로 주입, credentials 공유
- `FcmTokenUtils`: static 유틸 → Spring Bean 전환, 중복 credentials 로딩 제거

### Self-hosted Runner 설치 메모
- HP EliteBook 845 G7 (AMD Ryzen PRO) → **x64** runner
- Debian에서 `libicu-dev` 패키지 필요 (`apt install -y libicu-dev`)
- 데몬 등록: `sudo ./svc.sh install && sudo ./svc.sh start`
- runner 서비스 유저가 docker 그룹에 없으면 permission denied 발생
  - `sudo usermod -aG docker <runner-user>` 후 서비스 재시작

### 시트 앉는 자세
항상 모든 관절이 구부러진 채로 있어야 부상 위험 적음. + 편안한 자세 가능
- 왼발은 그냥 편하게 두셈.
- 오른발은 일직선으로 밟았을 때 무조건 브레이크가 밟히게 둬야 함. 위급상황 때는 몸이 자동으로 그냥 밟게 되어 있는데, 이렇게 세팅하지 않으면 잘못하면 엑셀 밟을 가능성이 큼. -> 엑셀은 45도 발이 틀어진 상태로.
	- 생각보다 발꿈치가 가까이 있는게 중요한듯? 그래야 발바닥 살로 컨트롤이 가능하니까?
- 뒷발은 무조건 고정된 상태로 있어야함. 
- 브레이크를 풀로 밟았을 때 무릎이 펴지면 절대 안됨. 사고 가능성 + 자세 불편
- 시트는 시야가 보일 만큼 위로 올려야 됨. 시야 확보 유리.
- 사이드미러가 잘 보일 만큼 뒤로 빼는것도 중요.

### 엑셀 브레이크 밟는 방법

패달 둘 다 눌렀다 땠다 반복하면 절 대 안 됨.
지긋이 누르고 지긋이 천천히 때야됨.

엑셀은 추가로 압력을 유지만 하고 있어도 -> 속도 선형적 증가.
기어가 자동으로 변속이 되면서 기어 단수가 계속 올라가기 때문에.
속도를 높이자 = 엑셀 압력을 살짝 더 높이자. 그리고 그 압력 유지하자.
속도를 낮추자 = 압력을 살짝 더 줄이자. 그 압력 유지하자.
속도 서서히 낮추면서 멈춰야 돼 = 엑셀 발을 때고 브레이크로 옮기면서 탄력 느끼면서 서서히 브레이크.

브레이크는 "발바닥 살"로 밟았다 땠다가 그 감각 익혀야 됨. -> 차 무게중심 쏠리지 않게 해야됨
0키로 되면 꾸욱 눌러서 적당한 압을 유지.

둘이 차이는 분명히 있음. 그래서 패달 밟는 자세도 브래이크를 되도록 일직선으로 두는 이유인듯. 컨트롤을 더 세부적으로 할 수 있어서.

RPM 컨트롤 필수. 

### 탄력 주행
차는 탄력 주행이다. 엑셀을 밟다가 때도 차는 그 탄력을 이용해서 계속 앞으로 나아간다.

신호등 등 차량을 멈춰야 되는 곳이 멀리서 보이기 시작? 혹은 마름모 사인 있다? 그럼 엑셀 밟다가 바로 브래이크 밟지 말고, 
1. 엑셀에서 발을 땜.
2. 브레이크로 발을 옮김.
3. 브레이크를 지긋이 밟기 시작함. 조절 시작.
이렇게 하는게 가장 자연스럽다.

### 주행차선 지키는 법
1. 차선의 중간 선을 임의로 머릿 속으로 그림.
2. 그 중간 선을 내 오른쪽 무릎 선과 맞춘다는 느낌으로 운전.
모든 차를 그렇게 운전하면 좋다고 함.

혹은
1. 왼쪽 차선이 본네트 위에 맞추는 느낌 혹은 끝에 겹치게.

후자는 경험해본 바로 조금 맞추기 힘들고 차종에 따라 다를 수도 있을듯. 우리는 중앙을 먼저 맞추는게 중요하므로. 하지만 양옆에 주차된 차로를 지나야 할 경우 등 차선을 걸쳐야 하는 경우도 있으니 결국 차폭의 감 자체를 아는것도 중요.

### 차 폭감 맞추는 법
그런거 없다. 반사 테이프 중간에 붙이고 하는데 결국 그냥 연습 많이 하는 수 밖에 없다.
직접 내려서 차폭감을 확인해보자. 

### 뒷바퀴는 캠퍼스의 축이다.
