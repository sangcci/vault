### 홈서버 구축 여정 정리
#### 문제점 & 해결
**[Cloud Image → ISO 설치]**
- 문제: Cockpit에서 Cloud Image 사용 시 `localhost login:` 진입 불가
- 원인: cloud-init이 패스워드 미설정
- 해결: .iso 파일로 직접 설치

**[sudo/vim 없음]**
- 문제: Debian 최소 설치 후 sudo, vim 없음
- 해결: `su -` 로 root 진입 후 설치, visudo로 sudoers 등록

**[DHCP + Static IP 공존]**
- 문제: 고정 IP 설정 후 DHCP IP가 secondary로 남아있음
- 원인: 기존 DHCP 임대가 만료 전
- 해결: `ip addr del` 로 수동 제거 또는 임대 만료 대기

**[iptables-persistent vs ufw 충돌]**
- 문제: iptables-persistent 설치 후 ufw 사라짐
- 원인: 두 패키지가 부팅 시 iptables 규칙 로드 역할 중복
- 해결: iptables-persistent 제거, ufw로 통일

**[포트 80 외부 접속 불가 - FORWARD chain DROP]**
- 문제: `ufw allow 80` 했는데 외부에서 VM nginx 접속 불가
- 원인: DNAT은 됐지만 FORWARD chain에서 ufw가 DROP
```
외부 패킷
    │
[PREROUTING] DNAT → 192.168.122.10:80  ✓
    │
[FORWARD]
    ufw-reject-forward → DROP  ✗
    우리 ACCEPT 규칙 (pkts: 0)  ← 여기까지 못 옴
```
- 해결: `ufw route allow proto tcp to 192.168.122.10 port 80`

**[ISP 포트 80 차단]**
- 문제: VM IP 직접 접속은 되는데 공인 IP/DuckDNS로 외부 접속 불가
- 원인: 한국 ISP 가정용 회선 80/443 포트 차단
- 해결: Cloudflare Tunnel 사용 (진행 중)

## Linux Kernel Networking 개념 정리

```
Linux Kernel Networking
│
├── Network Interface        ← 패킷 송수신 장치
│     ├── 물리 NIC           eth0, enp1s0, enx...
│     ├── 가상 브리지         virbr0 (KVM 기본)
│     └── 루프백             lo (127.0.0.1)
│
├── Routing Table            ← 경로 결정
│     "이 패킷 어느 인터페이스로?"
│     └── ip route
│
├── Netfilter / iptables     ← 패킷 필터링/변환
│     │
│     ├── Table
│     │     ├── nat          주소 변환 (DNAT/SNAT)
│     │     └── filter       허용/차단
│     │
│     └── Chain (패킷 관문)
│           ├── PREROUTING   들어올 때 (목적지 변경)
│           ├── INPUT        Host 자신이 목적지
│           ├── FORWARD      다른 곳으로 전달
│           ├── OUTPUT       Host에서 나갈 때
│           └── POSTROUTING  나갈 때 (출발지 변경)
│
├── ip_forward               ← 인터페이스 간 전달 스위치
│     0 = FORWARD 체인 동작 안함
│     1 = VM간 패킷 전달 가능
│
└── ufw                      ← iptables wrapper
      "사람이 쓰기 쉽게 감싼 도구"
      ufw allow   → INPUT 체인
      ufw route   → FORWARD 체인
```

**패킷 한 개의 전체 여정:**
```
외부 패킷 도착
      │
[물리 NIC 수신]
      │
[nat/PREROUTING]  목적지 주소 변경 (DNAT)
      │
[Routing Table]   어느 인터페이스로 보낼지 결정
      │
[filter/FORWARD]  통과 허용 여부 판단
      │
[nat/POSTROUTING] 출발지 주소 변경 (SNAT/MASQUERADE)
      │
[가상 NIC → VM]
```

**NAT vs Routing Table 차이:**
```
NAT           "주소 라벨 교체"    공인IP:80 → 192.168.122.10:80
Routing Table "경로 결정"        192.168.122.x → virbr0으로
```

**INPUT vs FORWARD 차이:**
```
목적지 = Host 자신  →  INPUT   (Cockpit 9090 등)
목적지 = VM        →  FORWARD  (nginx 80 등)

→ 9090은 ufw allow만으로 됐고
  80은 ufw route가 추가로 필요했던 이유
```

**Cloudflare Tunnel 원리:**
```
기존: 외부 → 공인IP:80 → VM  (포트 개방 필요)

Tunnel: VM → Cloudflare  (먼저 연결 수립)
        사용자 → Cloudflare → 기존 터널 재사용 → VM
        (포트 개방 불필요, ISP 차단 우회)
```

## Cockpit으로 Debian ISO VM 생성 전 과정

### 환경
- Host: HP Elitebook 845 G7 / Debian 13 / 16GB RAM / 512GB SSD
- 가상화: KVM + Cockpit

### Step 1. 필수 패키지 설치 (Host)
```bash
sudo apt install -y cockpit cockpit-machines virt-manager \
  qemu-kvm libvirt-daemon-system bridge-utils
sudo systemctl enable --now libvirtd
sudo systemctl enable --now cockpit.socket
```

### Step 2. ISO 다운로드 (Host)
- Debian 12 없음 → 13 (Trixie) 사용
- `debian-13.3.0-amd64-netinst.iso` (754MB) 선택
  - netinst = 최소 설치 이미지, 설치 중 인터넷으로 패키지 수신
```bash
sudo wget -P /var/lib/libvirt/images/ \
  https://cdimage.debian.org/debian-cd/current/amd64/iso-cd/debian-13.3.0-amd64-netinst.iso
```

### Step 3. Cockpit에서 VM 생성
```
브라우저 → https://localhost:9090
→ Virtual Machines → Create VM
→ Installation type: Local install media (ISO)
→ ISO 경로: /var/lib/libvirt/images/debian-13.3.0-amd64-netinst.iso
```

| VM | Name | Memory | Disk | vCPU |
|----|------|--------|------|------|
| Web | vm-web | 1024MB | 20GB | 1 |
| WAS | vm-was | 4096MB | 30GB | 2 |
| DB | vm-db | 4096MB | 100GB | 2 |
| Cache | vm-cache | 2048MB | 20GB | 1 |

### Step 4. Debian 설치 옵션
```
1. "Install" 선택 (Graphical Install 아님)
2. 언어: English
3. hostname: web-tier / was-tier / db-tier / cache-tier
4. 파티셔닝: Guided - use entire disk
5. Software selection:
   ☑ SSH server
   ☐ 나머지 전부 해제 (Desktop 등 불필요)
6. GRUB 설치: /dev/vda
```
- Cloud Image와 달리 ISO 설치는 직접 root 패스워드 설정 → `localhost login:` 문제 없음

### Step 5. 설치 후 기본 세팅 (VM 안에서)
```bash
# root로 sudo/vim 설치

# sudoers 등록
```

### Step 6. 고정 IP 설정 (VM 안에서)
```bash
# NIC 이름 확인
ip addr show
# → enp1s0 등으로 표시됨

sudo vim /etc/network/interfaces
```
```
auto enp1s0
iface enp1s0 inet static
  address 192.168.122.10   # vm-web 기준
  netmask 255.255.255.0
  gateway 192.168.122.1
  dns-nameservers 8.8.8.8
```
```bash
sudo systemctl restart networking
```
- 재시작 직후 DHCP IP가 secondary로 남을 수 있음 → 시간 지나면 자동 소멸
- 즉시 제거: `sudo ip addr del 192.168.122.x/24 dev enp1s0`

### Step 9. Docker 설치 (VM 안에서)
```bash
sudo apt remove -y docker docker.io containerd runc
sudo apt update && sudo apt install -y ca-certificates curl gnupg
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/debian/gpg | \
  sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg
echo "deb [arch=amd64 signed-by=/etc/apt/keyrings/docker.gpg] \
  https://download.docker.com/linux/debian trixie stable" | \
  sudo tee /etc/apt/sources.list.d/docker.list
sudo apt update
sudo apt install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin
sudo usermod -aG docker $USER && newgrp docker
```

### Step 10. Nginx 컨테이너 실행 (vm-web 안에서)
```bash
mkdir -p ~/nginx/conf.d

cat > ~/nginx/conf.d/default.conf <<'EOF'
server {
    listen 80;
    server_name _;
    location / {
        return 200 'vm-web nginx ok';
        add_header Content-Type text/plain;
    }
}
EOF

cat > ~/nginx/docker-compose.yml <<'EOF'
services:
  nginx:
    image: nginx:alpine
    container_name: nginx
    ports:
      - "80:80"
    volumes:
      - ./conf.d:/etc/nginx/conf.d:ro
    restart: unless-stopped
EOF

cd ~/nginx && docker compose up -d
curl http://localhost  # vm-web nginx ok
```

## Cloudflare Tunnel 메커니즘

**cloudflared가 먼저 Cloudflare에 아웃바운드 연결을 맺고 대기하는 구조**

```
[vm-web: cloudflared]  →→→→→→→→→→  [Cloudflare Edge]
        아웃바운드 연결 먼저 수립
        (QUIC/HTTP2, 포트 7844)

[사용자]  →  allreva.site  →  [Cloudflare Edge]  →  기존 세션 재사용  →  [nginx]
```

- 서버가 포트를 열 필요 없음 (inbound 없음)
- Cloudflare가 받은 요청을 기존 연결로 역방향 전달
- ISP 포트 80/443 차단 우회 (cloudflared는 7844 포트 사용)
- QUIC 프로토콜로 하나의 연결에서 여러 요청 멀티플렉싱
- 부팅 시 Cloudflare Edge 4개 PoP에 자동 연결 (Registered tunnel connection x4)

**Route (Public Hostname) vs DNS Record**

```
Public Hostname 설정 (Route)       "이 도메인 요청을 어느 서비스로?"
        ↓  저장 시 자동 생성        allreva.site → http://nginx:80
DNS Record (CNAME)                 "이 도메인의 주소는?"
        allreva.site → <tunnel-id>.cfargotunnel.com
```

- Route 없으면 터널이 살아있어도 목적지를 몰라 → Error 1016 발생
- `restart: unless-stopped` 설정 시 VM 재부팅 후 세션 자동 재수립
