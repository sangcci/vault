# Phenomenon: DB 이상 현상 (Anomaly)

## One-liner Definition
> (사전적) 데이터베이스 설계가 부적절할 때 데이터 중복으로 인해 데이터를 조작(삽입, 수정, 삭제)할 때 발생하는 부작용.
> (이해용) **장부 작성을 잘못해서 한 군데를 고쳤는데 다른 데는 그대로거나, 새로운 걸 적으려는데 필요 없는 정보까지 적어야 하는 꼬인 상황.**

---

## Observation (Context & Symptoms)
- **발생 환경**: 비정규화된 관계형 데이터베이스 테이블.
- **관찰 증상**: 
    1. **삽입 이상**: 새 데이터를 넣고 싶은데 특정 필수 값이 없어서 입력을 못 함.
    2. **갱신 이상**: 한 명의 주소를 바꿨는데 다른 곳엔 옛날 주소가 남아있음 (데이터 불일치).
    3. **삭제 이상**: 주문 정보를 지웠더니 고객 정보까지 같이 날아감.

---

## Hypothesized Causes
- 데이터 모델링 시 **함수적 종속성**을 고려하지 않은 무분별한 데이터 중복.
- 정규화 원칙 미준수.
