# 개념: 프로세스 스케줄러 (Process Scheduler)

## 📝 정의
CPU 자원을 여러 프로세스에 효율적으로 배분하기 위해, 다음에 실행할 프로세스를 선택하고 상태(Ready, Running, Waiting)를 전환하는 OS의 핵심 모듈입니다.

## ⚙️ 핵심 역할

### 1. 상태 관리 (State Management)
- **Ready**: 실행 준비가 완료되어 CPU 할당을 기다리는 상태.
- **Running**: 현재 CPU를 점유하여 명령을 수행 중인 상태.
- **Waiting (Sleep)**: I/O 작업이나 특정 이벤트(Lock 획득 등)를 기다리며 CPU를 반납한 상태.

### 2. 문맥 교환 수행
- 스케줄러는 프로세스 전환 시 레지스터 값, 스택 포인터 등 **PCB(Process Control Block)** 정보를 저장하고 복원합니다.

## 💡 본질과의 연결
- **`[[본질-추상화 (Abstraction)]]`**: 사용자나 개발자가 수많은 작업의 우선순위를 일일이 관리하지 않아도 되도록, OS 레이어에서 복잡한 자원 배분 로직을 숨겨줍니다.
- **`[[본질-동시성 (Concurrency)]]`**: 단일 코어에서도 스케줄러의 빠른 작업 교체를 통해 마치 여러 작업이 동시에 일어나는 것처럼 보이게 만듭니다.

## 🔗 관련 개념

- [[개념-차단 IO (Blocking IO)]] — 특정 이벤트 발생 시 스케줄러가 프로세스를 Waiting 상태로 보내는 원인
