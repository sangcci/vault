## 한 문장 정의

> (사전적) 오류가 발생한 프레임만 선택적으로 재전송하고, 나머지는 버퍼에 저장해 순서를 맞추는 ARQ.
> (이해용) 4번만 틀리면 4번만 다시 제출 — 나머지는 이미 받아두었다가 합침.

## 해결하는 문제

- Go-Back-N의 불필요한 재전송 문제 해결
- 고손실 환경에서의 전송 효율 극대화

## 치르는 비용

- 수신 측 **버퍼 필요** (순서 어긋난 프레임 임시 저장)
- 구현 복잡도 증가 (재정렬 로직)
- 윈도우 크기 제한이 Go-Back-N보다 엄격

## 동작 원리

```
윈도우 크기 = 4 (순서 번호: 0~7)

송신: 0─1─2─3─4─5...
             ↑ 2번 손실

수신: 0✓ 1✓ 2✗ 3✓(버퍼) 4✓(버퍼) 5✓(버퍼)
                ↑
     2번에 대해서만 NAK 또는 타임아웃 후 재요청

재전송: 2만 재전송

수신: 2✓ → 버퍼[3,4,5]와 합쳐서 순서대로 전달

[ 수신 윈도우 ]
Go-Back-N : 수신 윈도우 = 1 (버퍼 없음)
Sel. Repeat: 수신 윈도우 = N/2 (버퍼 필요)

[ 윈도우 크기 제한 ]
순서 번호 n비트 → 윈도우 크기 ≤ 2^(n-1)
```

## 실제 사용처

- **TCP SACK (Selective Acknowledgment)** — RFC 2018, 현대 TCP 표준
  - TCP 옵션 필드로 수신된 불연속 세그먼트 범위 알림
- **QUIC** — Google 개발, HTTP/3 기반
- 고손실 Wi-Fi, 모바일 환경 프로토콜
