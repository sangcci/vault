## 한 문장 정의

> (사전적) C 언어에서 하나의 표현식 안에 여러 연산자가 있을 때, 어떤 연산자를 먼저 실행할지를 결정하는 규칙.
> (이해용) 괄호를 치지 않아도 `a < b && c > d`가 `(a < b) && (c > d)`로 해석되는 이유.

---

## 해결하는 문제

- 복합 표현식에서 괄호 없이 실행 순서를 명확히 정의

---

## 치르는 비용

- 규칙을 모르면 의도와 다른 결과가 조용히 발생 (컴파일 에러 없음)

---

## 동작 원리

우선순위가 높을수록 먼저 결합한다.

```
높음 ──────────────────────────────────────────── 낮음

  !  ~  (단항)
  *  /  %
  +  -
  <  <=  >  >=        ← 비교 연산자
  ==  !=              ← 동등 연산자
  &   ^   |           ← 비트 연산자
  &&                  ← 논리 AND
  ||                  ← 논리 OR
  ?:
  =  +=  -=  ...      ← 대입 연산자
```

**핵심**: 비교 연산자(`<`, `>`, `==`)는 논리 연산자(`&&`, `||`)보다 우선순위가 높다.

```c
// 아래 두 표현식은 동일하다
a < b && c > d
(a < b) && (c > d)   // 비교가 먼저, 논리가 나중
```
