## 한 문장 정의

> (사전적) 응답률 `(대기 시간 + 서비스 시간) / 서비스 시간` 이 가장 높은 프로세스에 CPU를 할당하는 비선점 스케줄링.
> (이해용) SJF에 에이징을 내장한 버전 — 오래 기다릴수록 우선순위가 자동으로 오른다.

## 해결하는 문제

- SJF의 기아(Starvation) 문제 해결
- 짧은 작업 우선 + 오래 기다린 긴 작업도 결국 실행됨

## 치르는 비용

- 응답률 계산 비용 (스케줄링 결정마다)
- 실제 서비스 시간을 미리 알아야 함 → 실용적 구현 한계

## 동작 원리

```
응답률 = (대기 시간 + 서비스 시간) / 서비스 시간

프로세스  대기시간  서비스시간  응답률
P1        20        10        (20+10)/10 = 3.0
P2         5         5        (5+5)/5   = 2.0
P3        15        45        (15+45)/45 = 1.33

→ P1 선택 (응답률 3.0 최고)

[ 핵심: 대기 시간이 길수록 응답률 상승 ]
서비스=10인 P1이 30초 대기하면:
응답률 = (30+10)/10 = 4.0 → 더 높아짐
→ Aging 효과 내장
```

## 실제 사용처

- 주로 이론적·교육적 알고리즘
- SJF 기아 문제 해결 방안의 이론적 기준
