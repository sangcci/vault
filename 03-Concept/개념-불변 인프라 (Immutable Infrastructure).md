# 개념: 불변 인프라 (Immutable Infrastructure)

## 📝 정의
한 번 배포된 서버(또는 컨테이너)의 내부 상태를 절대로 수정하지 않는 운영 철학입니다. 변경이 필요하면 기존 서버를 수정하는 대신, 새로운 이미지로 새 서버를 띄우고 기존 서버를 폐기합니다.

---

## 🆚 대비되는 개념: 가변 인프라 (Mutable Infrastructure)
- **Snowflake Server (눈송이 서버)**: 관리자가 직접 접속해 패치하고 설정을 바꾸다 보면, 세상에 하나뿐인(재현 불가능한) 복잡한 서버가 되어버리는 현상.
- **문제점**: 서버가 망가졌을 때 똑같은 상태로 복구하기가 매우 어렵거나 불가능함.

---

## ✅ 불변 인프라의 장점
1.  **일관성**: 개발, 테스트, 운영 환경이 완벽히 동일함을 보장 (Docker Image).
2.  **신속한 복구**: 서버에 문제가 생기면 원인을 찾기보다 '새로 띄우기'로 즉시 해결.
3.  **버전 관리**: 인프라 상태를 코드로 관리(IaC)하여 이전 버전으로의 롤백이 용이함.

---

## 🛠️ 대표 도구
- **Docker**: 이미지 기반의 실행 환경 패키징.
- **Terraform / CloudFormation**: 선언적 인프라 프로비저닝.
