## 한 문장 정의

> (사전적) 준비 큐에서 CPU 버스트 시간이 가장 짧은 프로세스에게 먼저 CPU를 할당하는 비선점 스케줄링.
> (이해용) 계산대에서 물건 적은 손님을 먼저 처리. 긴 손님은 계속 밀릴 수 있다.

---

## 해결하는 문제

- 평균 대기 시간 최소화 — 동일 프로세스 집합에서 **이론적 최적** 알고리즘
- FCFS의 Convoy Effect 해소

---

## 치르는 비용

- **기아(Starvation)** — burst 시간이 긴 프로세스는 무한 연기 가능
- 실제 burst 시간을 미리 알 수 없음 → 실용적 구현 어려움 (예측 기반 근사)
- 선점이 안 되므로 긴 프로세스가 실행 중이면 짧은 프로세스도 대기

---

## 동작 원리

```
프로세스: P1(도착0, burst=8), P2(도착1, burst=4),
          P3(도착2, burst=2), P4(도착3, burst=5)

t=0: P1만 있으므로 P1 실행 (비선점, 끝까지)
t=8: 나머지 중 가장 짧은 P3(burst=2) 선택
t=10: P2(burst=4) 선택
t=14: P4(burst=5) 선택

Gantt Chart:
┌────────────────┬────┬────────┬──────────┐
│      P1        │ P3 │   P2   │    P4    │
└────────────────┴────┴────────┴──────────┘
0               8   10       14         19

평균 대기: P1=0, P3=6, P2=7, P4=11 → (0+6+7+11)/4 = 6

[ 기아 예시 ]
burst=100인 프로세스 + 계속 새 burst=1 프로세스 도착
→ burst=100은 영원히 실행 불가 (Aging으로 해결 → HRN)
```

## 실제 사용처

- 배치 시스템 (작업 시간 사전 알 때)
- 이론적 성능 비교 기준선 (실제 OS에선 미래 예측 불가)
