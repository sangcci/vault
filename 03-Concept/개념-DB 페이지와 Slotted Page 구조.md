## 한 문장 정의 (Definition)

> (사전적) 데이터베이스가 디스크 I/O를 수행하는 최소 물리적 단위인 페이지(Page) 내부에서, 가변 길이 레코드를 효율적으로 관리하기 위한 표준 물리 저장 구조.
> (이해용) 페이지라는 고정된 상자 안에 데이터를 무작위로 넣지 않고, 입구에 '슬롯(포인터)'을 두어 데이터의 실제 위치가 바뀌어도 외부(인덱스)에서 주소가 변하지 않게 관리하는 기술.

---

## 해결하는 문제 (Problem Solved)

- **가변 길이 레코드 관리**: 튜플의 크기가 제각각일 때 페이지 내부 공간을 낭비 없이 사용함.
- **주소 일관성 유지**: 페이지 내부에서 데이터가 이동(Compaction)하더라도 외부 인덱스가 가리키는 식별자(RID)를 수정할 필요가 없게 함. (**[[본질-간접 참조 (Indirection)]]**의 실현)

---

## 치르는 비용 (Cost/Trade-off)

- **공간 오버헤드**: 페이지마다 헤더와 슬롯 배열을 위한 추가 공간이 필요함.
- **관리 복잡도**: 레코드 삭제/수정 시 슬롯 배열과 데이터 영역 사이의 빈 공간을 관리(Fragmentation)해야 함.

---

## 동작 원리 (Mechanism)

### 1. 물리적 구조 시각화 (ASCII Diagram)
```text
+-------------------------------------------------------+
| Header: Page #, Free Space Pointer, Slot Count        |
+-------------------------------------------------------+
| Slot Array (Grow Downward)                            |
| [Slot 0: Offset 8000] [Slot 1: Offset 7950] [Slot 2]  | <-- RID(1,0)
+--------------------------+----------------------------+
|                          |                            |
|       (Free Space)       |        (Compaction)        |
|                          |                            |
+--------------------------+----------------------------+
|                          | [Record 2]                 |
+--------------------------+----------------------------+
|                          | [Record 1 (Offset 7950)]   |
+--------------------------+----------------------------+
| [Record 0 (Offset 8000)] (Data Area - Grow Upward)    |
+-------------------------------------------------------+
```

### 2. 간접 참조 메커니즘 (Indirection)
- **Record ID (RID)**: 외부 인덱스는 `(Page ID, Slot Number)`라는 주소를 가집니다.
- **데이터 이동 시**: 레코드가 페이지 내부에서 7950에서 7900으로 이동해도, `Slot 1`의 값만 `7900`으로 업데이트하면 됩니다. 외부 인덱스는 여전히 `Slot 1`을 가리키므로 아무런 수정이 필요 없습니다.

## 관련 본질 (Related Principles)

- **[[본질-논리 순서와 물리 순서는 다르다]]**: 논리적 식별자와 물리 배치가 독립일 수 있기에, 페이지 내부 이동과 외부 참조 안정성이 동시에 가능하다.
- **[[본질-간접 참조 (Indirection)]]**: 데이터의 물리적 위치와 논리적 식별자를 분리하는 핵심 원리.
- **[[본질-최소 단위의 불일치 (Granularity Mismatch)]]**: DB 페이지와 OS 페이지 크기 차이로 발생하는 물리적 이슈.
