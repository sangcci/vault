# 개념: DB 페이지와 Slotted Page 구조

## 📝 정의
DB가 디스크와 데이터를 주고받는 최소 물리적 단위는 **페이지(Page)**이며, 그 내부에서 다양한 크기의 데이터를 효율적으로 관리하기 위해 사용하는 구조가 **Slotted Page**입니다.

## ⚙️ 물리적 구조: Slotted Page
페이지 내부는 크게 세 영역으로 나뉩니다.

1. **Header**: 페이지 번호, 크기, 빈 공간의 위치 등 메타데이터를 담습니다.
2. **Slot Array (인덱스)**: 실제 데이터가 페이지 내의 어느 위치(Offset)에 있는지 기록하는 포인터 배열입니다. (앞에서 뒤로 증가)
3. **Data (레코드)**: 실제 튜플 데이터가 저장되는 공간입니다. (뒤에서 앞으로 증가)

> **왜 이렇게 배치하나?**: 인덱스와 데이터가 서로 마주 보며 자라나게 함으로써, 페이지 내부 공간을 남김없이 유연하게 쓰기 위함입니다.

## ⚙️ 식별자: Record ID (RID)
데이터베이스의 특정 행(Row)을 가리키는 고유 주소표입니다.
- **구성**: `(Page ID, Slot Number)`
- **특징**: 데이터가 페이지 내부에서 위치를 옮겨도(Compaction), 슬롯 번호만 유지하면 외부 인덱스는 수정할 필요가 없습니다. (간접 참조의 이점)

## 💡 본질과의 연결
- **`[[본질-최소 단위의 불일치 (Granularity Mismatch)]]`**: DB 페이지(주로 8KB/16KB)는 OS 페이지(4KB)보다 큽니다. 이 차이 때문에 발생하는 **Torn Page** 현상을 방어하는 것이 DB 엔진의 숙제입니다.
- **I/O 효율성**: 행 하나를 읽더라도 페이지 전체를 읽는 이유는 디스크의 **Sequential Access** 이점을 극대화하기 위해서입니다.
