## 한 문장 정의

> (사전적) 오류가 발생한 프레임부터 윈도우 내 모든 프레임을 재전송하는 Sliding Window 기반 ARQ.
> (이해용) 4번 틀리면 4번부터 끝까지 전부 다시 제출.

## 해결하는 문제

- Stop-and-Wait의 낮은 회선 이용률 개선
- 파이프라인 방식으로 여러 프레임 연속 전송

## 치르는 비용

- 오류 1개 발생 시 윈도우 내 **이후 프레임 전부 재전송** → 불필요한 재전송 비용
- 수신 버퍼 1개로 충분 (순서 맞지 않는 프레임 버림)

## 동작 원리

```
윈도우 크기 = 4 (순서 번호: 0~7)

송신: 0─1─2─3─ (4─5─6... 윈도우 밖)
             ↑ 오류 발생 (2번 손실)

수신: 0✓ 1✓ 2✗ → 3,4,5 NAK/무시 (순서 어긋남)

재전송: 2─3─4─5─ (2부터 전부 재전송)

[ 수신 측 동작 ]
순서 맞는 프레임만 수락, 나머지는 버림(Discard)
→ 수신 버퍼 = 1개로 충분

[ 윈도우 크기 제한 ]
순서 번호 n비트 → 윈도우 크기 ≤ 2^n - 1
(2^n으로 하면 모든 재전송 시 ACK와 구분 불가)
```

## 실제 사용처

- 초기 ARQ 구현
- 일부 무선 링크 계층 프로토콜
- 현대 TCP는 Selective Repeat(SACK) 방식 사용
