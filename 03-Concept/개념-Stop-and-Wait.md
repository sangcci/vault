## 한 문장 정의

> (사전적) 송신 측이 프레임 하나를 보내고 ACK를 수신할 때까지 다음 전송을 멈추는 ARQ 프로토콜.
> (이해용) 문자 보내고 읽었다는 답장 올 때까지 다음 문자 안 보내기.

## 해결하는 문제

- 가장 단순한 신뢰성 있는 데이터 전송 보장
- 구현 복잡도 최소화

## 치르는 비용

- **낮은 회선 이용률** — 전파 지연 동안 유휴 상태
  - 이용률 = T_frame / (T_frame + 2 × T_prop)
- RTT가 클수록 효율 급락 (위성 통신에서 최악)

## 동작 원리

```
송신                    수신
  │──── Frame 0 ────→  │
  │                    │ (처리)
  │  ←──── ACK 0 ────  │
  │──── Frame 1 ────→  │
  │                    │
  │  ←──── ACK 1 ────  │

[ 회선 이용률 ]
T_frame = 1ms, RTT = 20ms
이용률 = 1 / (1 + 20) ≈ 4.7% ← 대역폭 낭비

[ 오류 처리 ]
타임아웃 → 재전송 (ACK 유실 시 중복 수신 가능)
→ Sliding Window로 해결
```

## 실제 사용처

- 레거시 프로토콜 (Xmodem)
- 일부 임베디드 제어 시스템 (지연 허용, 단순성 중요)
- [[개념-Sliding Window]] 및 [[개념-Go-Back-N]] 이해의 기준선
