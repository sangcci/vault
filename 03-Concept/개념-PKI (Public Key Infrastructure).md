## 한 문장 정의

> (사전적) 공개키 암호화를 기반으로 디지털 인증서를 발급·검증·폐기하는 신뢰 체계.
> (이해용) **"신뢰할 수 없는 인터넷에서 상대방의 신원을 제3자 보증으로 검증하는 구조."** 여권을 정부가 발급하고 국경에서 검증하는 것과 동일한 원리.

---

## 해결하는 문제

- 처음 만나는 서버/사용자가 진짜인지 확인할 수단이 없음 (중간자 공격 위험)
- 공개키만으로는 "이 키가 누구의 것인지" 증명 불가

---

## 치르는 비용

- CA 인증서 발급 및 갱신 관리 부담
- 인증서 만료 시 서비스 중단 위험
- CA 자체가 침해당하면 신뢰 체계 전체 붕괴 (DigiNotar 사례)

---

## 동작 원리

### 1. 핵심 구성요소

```
Root CA (최상위 인증기관)
  └── Intermediate CA (중간 인증기관)
        └── End-Entity Certificate (서버/사용자 인증서)
```

- **CA (Certificate Authority)**: 인증서를 발급·서명하는 신뢰 기관
- **인증서**: 공개키 + 소유자 정보 + CA 서명이 묶인 문서 (X.509 형식)
- **신뢰 체인 (Chain of Trust)**: Root CA → Intermediate CA → 인증서로 이어지는 서명 검증 경로

---

### 2. 전자서명 메커니즘 (Hash + RSA)

PKI의 모든 서명은 [[개념-전자서명 (Digital Signature)]] 메커니즘으로 동작한다.
CA가 인증서에 서명할 때, 사용자가 거래에 서명할 때 모두 동일한 Hash + RSA 조합을 사용한다.

---

### 3. HTTPS 인증서 검증 흐름

```
브라우저                          서버
    │                               │
    │──── ClientHello ─────────────→│
    │←─── 서버 인증서 (cert) ────────│
    │
    │  [검증 과정]
    │  1. cert 안의 CA 서명 추출
    │  2. 브라우저 내장 Root CA 공개키로 복호화 → H'
    │  3. hash(cert 내용) → H
    │  4. H == H' → 인증서 신뢰
    │  5. cert 안의 서버 공개키로 세션키 암호화
    │
    │──── 암호화된 세션키 ──────────→│
    │        (이후 AES 대칭키 통신)
```

- 브라우저는 Root CA 목록을 OS/브라우저에 내장하여 신뢰 기반으로 삼는다.

---

### 4. 공동인증서 (사용자 서명) 흐름

```
사용자                              은행 서버
    │                                   │
    │  [서명]                           │
    │  hash(거래내용) → H               │
    │  RSA(H, 사용자 private key)       │
    │         → signature               │
    │                                   │
    │ ──── 거래내용 + signature ──────→ │
    │                                   │  [검증]
    │                                   │  사용자 public key로 복호화 → H'
    │                                   │  hash(거래내용) → H
    │                                   │  H == H' → 본인 확인
```

- HTTPS와 구조 동일, 서명 주체만 다름 (CA → 사용자)

---

### 5. 두 맥락 비교

```
              HTTPS                  공동인증서
서명자:       CA                     사용자 본인
검증자:       브라우저               은행/정부 서버
증명 대상:    서버 신원               사용자 의사 (부인방지)
인증서 발급:  Let's Encrypt 등       금융결제원 등
```
