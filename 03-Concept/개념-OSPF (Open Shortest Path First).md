## 한 문장 정의

> (사전적) 링크 상태 정보를 AS 내 모든 라우터와 공유해 최단 경로를 계산하는 동적 라우팅 프로토콜.
> (이해용) 모든 라우터가 네트워크 전체 지도를 가지고 각자 최단 경로를 계산하는 방식.

## 해결하는 문제

- RIP의 홉 수 제한과 느린 수렴 문제
- 대규모 네트워크에서의 정확한 경로 계산

## 치르는 비용

- 모든 라우터가 전체 토폴로지 DB를 유지 → 메모리·CPU 사용량 증가
- 초기 설정과 구성이 RIP보다 복잡

## 동작 원리

**링크 상태 방식 — 전체 토폴로지 공유**
```
각 라우터가 LSA(Link State Advertisement) 생성
  → "나는 A와 연결됨, 비용 10 / B와 연결됨, 비용 5"
LSA를 AS 내 모든 라우터에 플러딩
→ 모든 라우터가 동일한 토폴로지 지도 보유
```

**알고리즘: 다익스트라 (Dijkstra)**
```
전체 지도를 보유하고 있으므로
각 라우터가 자신을 루트로 최단 경로 트리 계산
→ RIP보다 정확하고 빠른 수렴
```

**다익스트라 과정**
```
네트워크 (A 기준):
  A -1- B
  A -4- C
  B -2- C
  B -5- D
  C -1- D

초기: A=0, B=∞, C=∞, D=∞
미방문: {A, B, C, D}

Step 1: 비용 최소 노드 A 선택
  A → B 갱신: 0+1 = 1  ✓
  A → C 갱신: 0+4 = 4  ✓
  미방문: {B, C, D}

Step 2: 비용 최소 노드 B(1) 선택
  B → C 갱신: 1+2 = 3  < 4  ✓ 갱신
  B → D 갱신: 1+5 = 6  ✓
  미방문: {C, D}

Step 3: 비용 최소 노드 C(3) 선택
  C → D 갱신: 3+1 = 4  < 6  ✓ 갱신
  미방문: {D}

Step 4: D(4) 선택 → 완료

최종: A=0, B=1, C=3, D=4
→ 토폴로지 변경 시 LSA 플러딩 후 즉시 재계산
```

**RIP vs OSPF 비교**
```
항목          RIP                  OSPF
방식          거리 벡터            링크 상태
알고리즘      벨만-포드            다익스트라
메트릭        홉 카운트            대역폭(비용)
최대 규모     15홉 제한            제한 없음
수렴 속도     느림                 빠름
적합 환경     소규모 단순 네트워크  중·대규모 네트워크
```
