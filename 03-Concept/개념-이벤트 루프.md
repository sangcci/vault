# Concept: 이벤트 루프

## 한 문장 정의
> (사전적) 시스템에서 발생하는 이벤트(I/O, 타이머 등)를 대기하고, 이벤트가 발생하면 등록된 콜백을 실행하는 무한 루프.
> (이해용) **은행 창구의 번호표 기계** (손님은 번호표만 뽑고 대기하다가, 딩동 소리가 나면 창구로 감).

## 해결하는 문제
- **Blocking I/O의 비효율성**: 파일 읽기/네트워크 요청이 끝날 때까지 CPU가 노는 문제(Busy Waiting)를 해결.
- **C10K 문제**: 스레드를 무한정 늘리지 않고도 수만 개의 동시 접속을 처리.

## 치르는 비용
- **CPU Bound 작업에 취약**: 루프 자체가 막히면 모든 요청이 멈춤 (Head-of-Line Blocking).
- **디버깅 어려움**: 콜백 지옥(Callback Hell)이나 비동기 흐름 추적의 난해함.

## 동작 원리 (Mechanism)
1. **Event Queue**: 발생한 이벤트들이 줄을 섬.
2. **Loop**: 큐를 계속 감시하다가 이벤트가 있으면 꺼냄.
3. **Handler**: 해당 이벤트에 연결된 함수(Callback)를 실행.
   - *핵심 기술*: `epoll` (Linux), `kqueue` (BSD) 같은 OS 커널의 알림 기능 활용.
