## 한 문장 정의

> (사전적) 비대칭 키 암호화를 이용해 데이터의 무결성과 서명자 인증을 동시에 보장하는 암호화 기법.
> (이해용) **"내용이 바뀌지 않았고, 내가 직접 서명했음을 수학적으로 증명하는 것."** 도장이 위조 불가능해진 버전.

## 해결하는 문제

- 데이터가 전송 중 변조되었는지 알 수 없음 (무결성)
- 데이터를 보낸 사람이 정말 그 사람인지 알 수 없음 (인증)
- 나중에 "나는 서명한 적 없다" 주장을 막을 수 없음 (부인방지)

## 치르는 비용

- RSA 연산이 느려서 긴 메시지 직접 서명 불가 → 반드시 Hash 선행 필요
- private key 유출 시 모든 서명이 위조 가능

## 동작 원리

### 서명 + 검증 흐름

```
[서명]
서명자: hash(데이터) ──→ H
        RSA(H, private_key) ──→ signature

[검증]
검증자: RSA(signature, public_key) ──→ H'
        hash(데이터) ──→ H
        H == H' → 서명 유효
```

- **hash**: 데이터 무결성 보장 + RSA 입력 크기 고정
- **RSA**: private key 소유자만 서명 생성 가능 → 인증 + 부인방지

### 위조가 불가능한 이유

```
공격자 목표: (M, signature) → (M', signature') 교체

signature' = RSA(hash(M'), private_key)
                               ↑
                    private_key 없이 생성 불가
                    public_key → private_key 역산 = 소인수분해 문제

공격 유형                   결과
────────────────────────────────────────
M만 변조                    hash 불일치 → 검증 실패
signature만 변조            복호화값 ≠ hash(M) → 검증 실패
M + signature 동시 변조     private key 없으면 위조 불가
```

### Hash 충돌 저항성

```
SHA-256 충돌 탐색 (생일 공격 기준):
시도 횟수 2^128 / 슈퍼컴퓨터 10^18회/초
= 10^20년 ≈ 우주 나이의 10^10배
```

보안 근거는 "해시가 같다"는 사실이 아니라 **같은 해시를 만드는 다른 데이터를 찾는 계산 비용이 현실적으로 불가능**하다는 것.
→ MD5, SHA-1은 충돌이 실제 발견되어 폐기됨.

## 전자서명 vs PKI 구분

```
전자서명 (메커니즘)          PKI (인프라)
─────────────────────────────────────────────
"어떻게 서명하는가"          "누가 키를 보증하는가"
Hash + RSA 동작 방식         CA, 인증서, 신뢰 체인 관리
도구 (Tool)                  체계 (System)
PKI 없이도 존재 가능          전자서명을 도구로 사용
```

- 전자서명은 PKI의 **구성 요소**
- PKI 없는 전자서명: PGP (CA 없이 개인 간 직접 신뢰)
- PKI는 전자서명에 **"누가 서명했는가"의 신뢰 근거**를 추가
