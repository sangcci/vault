## 한 문장 정의

> (사전적) 참조 비트(R bit)를 이용해 LRU를 근사하는 알고리즘. 참조된 페이지에 "한 번의 기회"를 더 주고 시계 방향으로 순환한다.
> (이해용) 한 바퀴 돌면서 최근에 쓴 흔적(R=1)이 있으면 넘어가고 흔적 없는 것(R=0)을 교체.

## 해결하는 문제

- 완전 LRU의 타임스탬프 관리 비용 없이 LRU에 근접한 성능
- 하드웨어 접근 비트(Access bit) 하나만으로 구현 가능
- **실제 Linux 커널이 사용하는 알고리즘**

## 치르는 비용

- 완전 LRU보다 Page Fault 약간 더 많을 수 있음 (근사이므로)
- 포인터 순환 오버헤드

## 동작 원리

```
[ 원형 큐 + 참조 비트(R) ]

    ↓ 포인터(교체 후보)
┌─────┬─────┬─────┬─────┬─────┐
│P1 R=1│P2 R=0│P3 R=1│P4 R=0│P5 R=1│
└─────┴─────┴─────┴─────┴─────┘

교체 필요 시:
1. 포인터가 가리키는 페이지의 R 확인
   - R=1 이면: R=0 으로 초기화 후 다음으로 이동 (Second Chance 부여)
   - R=0 이면: 해당 페이지 교체

[ 예시 순서 ]
P1: R=1 → R=0으로 초기화, 포인터 이동
P2: R=0 → 교체! ← 기회를 이미 소진한 페이지

[ 참조 비트 갱신 ]
페이지 참조 시 R=1로 자동 세팅 (하드웨어 MMU가 처리)
주기적으로 R 비트 전체 초기화 (시계 1바퀴 = 1주기)
```

## 실제 사용처

- **Linux kernel** — `mm/vmscan.c` active/inactive list (2-hand clock)
- Unix 계열 OS 대부분의 실제 페이지 교체 구현
- 완전 LRU보다 구현 비용 낮아 대부분 실용 시스템에서 선택
