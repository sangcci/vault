## 한 문장 정의

> (사전적) 현재 실행 중인 프로세스의 남은 시간과 새로 도착한 프로세스의 burst 시간을 비교해, 더 짧은 쪽에 CPU를 할당하는 선점형 SJF.
> (이해용) 계산대에서 줄 선 손님보다 물건 더 적은 손님이 오면 즉시 교체.

## 해결하는 문제

- SJF의 비선점 한계 보완 — 새 프로세스 도착 시 즉시 재스케줄링
- 시분할(Time-sharing) 시스템에서 응답성 향상

## 치르는 비용

- 각 프로세스의 남은 실행 시간을 추적해야 함 → **오버헤드 증가**
- 잦은 선점으로 Context Switch 비용 상승
- 기아(Starvation) 여전히 존재

## 동작 원리

```
P1(도착0, burst=8), P2(도착1, burst=4),
P3(도착2, burst=2), P4(도착3, burst=5)

t=0: P1 실행 (잔여 8)
t=1: P2 도착 (burst=4) < P1 잔여(7) → P1 선점, P2 실행
t=2: P3 도착 (burst=2) < P2 잔여(3) → P2 선점, P3 실행
t=3: P4 도착 (burst=5) > P3 잔여(1) → 선점 없음
t=4: P3 완료 → P2 잔여(3) vs P1 잔여(7) vs P4(5) → P2 실행
...

Gantt Chart:
┌──┬──┬──┬──────┬──────────┬────────────────┐
│P1│P2│P3│  P2  │    P4    │       P1       │
└──┴──┴──┴──────┴──────────┴────────────────┘
0  1  2  3     4           7              15

[ SJF vs SRT 비교 ]
SJF  : 비선점, 시작하면 완료까지 실행
SRT  : 선점형, 더 짧은 잔여 시간 도착 시 즉시 교체
```

## 실제 사용처

- 이론적 알고리즘 (실제 OS는 MLFQ/CFS 사용)
- 시분할 시스템 분석의 기준 모델
