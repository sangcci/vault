## 한 문장 정의

> (사전적) 그래프에서 시작 노드로부터 가까운 노드를 먼저 탐색하는 알고리즘.
> (이해용) 물이 퍼지듯 거리 순서대로 탐색한다. 먼저 발견한 것이 항상 최단 거리다.

## 해결하는 문제

- 간선 가중치가 균등할 때의 최단 경로 탐색
- 암묵적 그래프(상태 공간)에서의 최소 이동 횟수

## 치르는 비용

- 메모리: 큐와 visited 배열로 O(V) 공간 필요
- 가중치가 불균등하면 최단 경로 보장 불가 → Dijkstra 필요

## 동작 원리

```
시작 노드를 큐에 삽입, visited 표시
while 큐가 비어있지 않음:
    node = 큐에서 꺼냄          ← O(1), FIFO
    for neighbor in node의 인접 노드:
        if not visited[neighbor]:
            visited[neighbor] = True
            큐에 삽입
```

**시간복잡도: O(V + E)**

```
큐 삽입 횟수 = 각 노드당 최대 1번   → O(V)
간선 확인 횟수 = 각 간선당 최대 1번  → O(E)
─────────────────────────────────────
총 연산 = O(V + E)
```

visited 배열이 재방문을 막으므로, 모든 노드와 간선을 정확히 1번씩만 처리한다.

**왜 log N이 붙지 않는가**

```
일반 큐 (FIFO):
  삽입 → 맨 뒤에 추가  O(1)
  삭제 → 맨 앞에서 제거 O(1)
  → 정렬 불필요

힙 (우선순위 큐):
  삽입 → 부모와 비교하며 올라감  O(log N)
  삭제 → 아래로 내려가며 재정렬 O(log N)
  → E번 삽입 × log V = O(E log V)
```

BFS는 "먼저 발견 = 최단 거리"가 간선 가중치 균등으로 자동 성립하므로, 순서를 바꿀 이유가 없다. 정렬이 필요 없으니 log가 없다.

**숨바꼭질 (BOJ 1697) 적용 예시**

```
상태: 현재 위치 x (0 ~ 100000)
간선: x-1, x+1, 2x (각 1초 = 균등 가중치)

V = 100001, E ≤ 3 × 100001
→ O(V + E) = O(N) ≈ 10만 연산
```
