## 한 문장 정의

> (사전적) 사용자 프로세스가 커널의 서비스를 요청하기 위해 호출하는 **OS가 제공하는 API**.
> (이해용) 식당에서 손님(사용자 프로세스)이 주방(커널)에 직접 들어갈 수 없으니, 주문서(시스템 콜)를 통해 요리(하드웨어 접근, 프로세스 생성 등)를 요청하는 것.

---

## 해결하는 문제

- 사용자 프로세스는 자신의 주소 공간 밖 메모리 접근, 하드웨어 제어, 프로세스 생성 등을 직접 수행할 수 없음
- 커널이 이 작업들을 대행하되, 정해진 인터페이스(API)로만 요청을 받아 보안과 안정성 확보

---

## 치르는 비용

- **모드 전환 오버헤드**: User Mode → Kernel Mode 전환 시 CPU 특권 레벨 변경, 레지스터 저장 등 발생
- **OS마다 API가 다름**: Windows(CreateProcess)와 Unix(fork+exec)는 호환 불가 → POSIX 같은 표준이 필요한 이유

---

## 동작 원리

### 시스템 콜 = API

시스템 콜의 본질은 **API**. 커널이 제공하는 함수 목록이며, C/C++ 함수 시그니처로 노출됨.

```text
┌──────────────────────────────────────────────┐
│  사용자 프로세스                                │
│    write(fd, buf, len);   ← C 함수 호출       │
│              │                                │
│              ↓ (트랩/인터럽트)                   │
├──────────────────────────────────────────────┤
│  커널                                         │
│    sys_write() 실행      ← 실제 하드웨어 접근   │
│              │                                │
│              ↓ (완료)                           │
├──────────────────────────────────────────────┤
│  사용자 프로세스                                │
│    write() 반환           ← 결과 수신          │
└──────────────────────────────────────────────┘
```

핵심: **User Space에서 호출, Kernel Space에서 실행, 다시 User Mode로 복귀**.

### 주요 분류

| 분류 | 예시 | 설명 |
|------|------|------|
| **프로세스 관리** | fork(), execve(), wait(), exit() | 프로세스 생성·교체·종료 |
| **파일 IO** | open(), read(), write(), close() | 디스크 파일 접근 |
| **장치 IO** | ioctl(), mmap() | 하드웨어 장치 제어 |
| **정보 조회** | getpid(), uname() | 시스템/프로세스 정보 |
| **IPC** | pipe(), shmget(), socket() | 프로세스 간 통신 채널 |

### User Space vs Kernel Space — 왜 구분하는가

```text
사용자 프로세스가 직접 하드웨어를 제어할 수 있다면:
  → 악성 프로세스가 디스크 전체를 지울 수 있음
  → 한 프로세스가 다른 프로세스의 메모리를 읽을 수 있음
  → 시스템 전체가 불안정해짐

해결: CPU가 실행 모드를 하드웨어 레벨에서 분리
  → User Mode: 제한된 명령어만 실행 가능
  → Kernel Mode: 모든 명령어 실행 가능
  → 모드 전환은 시스템 콜(트랩)을 통해서만 허용
```

이 구분에 대한 상세 내용: [[탐구-커널과 운영체제의 경계는 어디인가]]

### 모드 전환 vs 문맥 교환

| | 모드 전환 (Mode Switch) | 문맥 교환 (Context Switch) |
|--|------------------------|--------------------------|
| **대상** | 같은 프로세스 내에서 특권 레벨 변경 | 다른 프로세스로 실행 주체 변경 |
| **발생 시점** | 시스템 콜 호출 시 | 스케줄러가 CPU 재할당 시 |
| **비용** | 상대적으로 가벼움 | TLB Flush, 캐시 미스 등 무거움 |
| **관계** | 문맥 교환에는 모드 전환이 포함됨 | 모드 전환이 항상 문맥 교환은 아님 |
