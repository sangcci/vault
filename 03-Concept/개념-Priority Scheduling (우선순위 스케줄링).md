## 한 문장 정의

> (사전적) 각 프로세스에 우선순위를 부여하고, 가장 높은 우선순위 프로세스에 CPU를 할당하는 스케줄링.
> (이해용) 응급실 — 위급도에 따라 순서를 재배치, 경증 환자는 계속 밀릴 수 있다.

---

## 해결하는 문제

- 중요도 차이가 있는 작업 환경 — 시스템 프로세스, 실시간 태스크 우선 처리
- 인터럽트 처리 우선순위 계층 구현

---

## 치르는 비용

- **기아(Starvation)** — 낮은 우선순위 프로세스는 무한 대기 가능
  - 해결: **Aging** — 대기 시간에 비례해 우선순위 점진적 상승
- 선점형/비선점형 모두 구현 가능

---

## 동작 원리

```
[ 비선점형 ]
프로세스: P1(pr=3), P2(pr=1, 최고), P3(pr=4), P4(pr=2)
(숫자 작을수록 높은 우선순위)

대기 큐: [P2(1), P4(2), P1(3), P3(4)]
실행 순서: P2 → P4 → P1 → P3

[ 선점형 — 더 높은 우선순위 도착 시 즉시 교체 ]
t=0: P1(pr=3) 실행 중
t=2: P2(pr=1) 도착 → P1 선점, P2 실행

[ Aging으로 기아 해결 ]
대기 1초마다 우선순위 1씩 증가:
P3(pr=4, 대기 4초) → pr=0 (최고 우선순위로 상승)
```

---

## 실제 사용처

- **Linux 커널** — `task_struct.prio` (0=최고, 139=최저), 실시간 태스크 0~99
- **RTOS** (QNX, VxWorks) — 하드 실시간 시스템
- **Windows 스레드 우선순위** — 0~31 레벨
- **Kubernetes QoS** — Guaranteed > Burstable > BestEffort 순 선점
- **네트워크 QoS** — 패킷 우선순위 처리 (DSCP)
