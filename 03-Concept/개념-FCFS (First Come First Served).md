## 한 문장 정의

> (사전적) 준비 큐에 도착한 순서대로 CPU를 할당하는 비선점 스케줄링 알고리즘.
> (이해용) 줄 선 순서대로 처리하는 은행 창구. 앞에 느린 고객이 있으면 뒤가 다 기다린다.

## 해결하는 문제

- 구현이 단순한 스케줄링이 필요한 배치 처리 환경
- 공정성(순서 보장) — 기아(starvation) 없음

## 치르는 비용

- **Convoy Effect** — 실행 시간이 긴 프로세스 뒤에 짧은 프로세스들이 대기
- 평균 대기 시간이 길어질 수 있음 (최적이 아님)
- 대화형(interactive) 시스템에 부적합

## 동작 원리

```
도착 순서: P1(burst=8) → P2(burst=2) → P3(burst=4)

Gantt Chart:
┌────────────────┬────┬────────┐
│      P1        │ P2 │   P3   │
└────────────────┴────┴────────┘
0               8   10       14

대기 시간: P1=0, P2=8, P3=10
평균 대기: (0+8+10)/3 = 6

[ Convoy Effect ]
P1(burst=100) → P2(burst=1) → P3(burst=1)
P2, P3는 100초 대기 → CPU 낭비
```

## 실제 사용처

- 배치 처리 시스템 (Batch Processing)
- 프린터 스풀러
- 디스크 I/O 큐 (단순 구현 시)
