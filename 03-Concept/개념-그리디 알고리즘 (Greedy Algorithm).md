## 한 문장 정의

> (사전적) 각 단계에서 지역 최적 선택을 반복해 전체 최적해를 구하려는 알고리즘 패러다임.
> (이해용) "지금 당장 가장 좋아 보이는 것을 선택하고, 뒤를 돌아보지 않는 전략."

---

## 해결하는 문제

- 최적화 문제에서 완전 탐색(Brute Force)의 지수 시간 복잡도를 줄임

---

## 치르는 비용

- 최적해를 항상 보장하지 않음 — 문제 구조가 맞지 않으면 틀린 답을 냄

---

## 동작 원리

그리디가 정당한지 판단하는 두 가지 조건:

```
1. 그리디 선택 속성 (Greedy Choice Property)
   지역 최적 선택이 전역 최적해에 포함된다.

2. 최적 부분 구조 (Optimal Substructure)
   선택 후 남은 부분 문제의 최적해가 전체 최적해에 포함된다.
```

**증명 기법 — Exchange Argument:**
> "내 선택이 최적이 아니라고 가정하자. 최적해의 선택과 바꿔도 결과가 나빠지지 않음을 보이면, 내 선택도 최적이다."

**예시 — Interval Scheduling (활동 선택 문제):**

```
활동:  A[1,3]  B[2,5]  C[3,6]  D[5,7]
전략:  종료 시간이 가장 빠른 활동부터 선택

step1: A[1,3] 선택 → B[2,5] 제거 (겹침)
step2: C[3,6] 제거 (A와 겹침)
step3: D[5,7] 선택

결과: {A, D} → 최대 2개 (최적)
```

종료 시간 기준 정렬 후 순서대로 선택하면 항상 최대 개수가 보장됨.
Exchange Argument로 증명 가능 — 다른 선택으로 바꿔도 개수가 줄거나 같다.
