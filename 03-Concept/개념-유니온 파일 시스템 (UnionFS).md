# 개념: 유니온 파일 시스템 (UnionFS)

## 📝 정의
여러 개의 디렉토리를 하나로 통합하여, 마치 하나의 파일 시스템처럼 보이게 만드는 기술입니다. Docker 이미지가 여러 개의 '레이어'로 구성될 수 있는 물리적 근거입니다.

## ⚙️ 핵심 원리
1.  **Read-Only Layers**: 베이스 이미지(OS, 런타임 등)는 수정 불가능한 상태로 겹쳐 쌓임.
2.  **Writable Layer**: 가장 최상단에만 쓰기 가능한 얇은 층을 두어 앱 실행 중 발생하는 변경사항을 저장함.
3.  **Copy-on-Write (CoW)**: 하위 레이어의 파일을 수정해야 할 때, 원본을 건드리지 않고 최상단 쓰기 레이어로 복사해온 뒤 수정함.

## ✅ 해결하는 문제
- **저장 공간 절약**: 동일한 베이스 이미지를 쓰는 100개의 컨테이너가 있어도 실제 디스크에는 베이스 레이어가 한 번만 저장됨.
- **배포 속도**: 변경된 레이어(소스 코드 등)만 전송하면 되므로 배포가 매우 빠름.
