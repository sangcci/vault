## 한 문장 정의 (Definition)

> 실제 외부 서버 대신 가상의 HTTP 서버를 띄워, 미리 정의된 응답(Stub)을 반환하게 함으로써 외부 API 의존성을 격리하는 테스트 기술.

---

## 1. 개요 (Overview)

현대적인 시스템은 수많은 외부 API(결제, 인증, 날씨 등)와 통신합니다. 테스트 시 실제 API를 호출하면 네트워크 상태, 호출 제한(Rate Limit), 데이터 오염 등의 문제로 테스트가 불안정해집니다. 이를 해결하기 위해 **스터빙(Stubbing)** 기술을 사용하여 테스트 환경을 통제합니다.

---

## 2. 핵심 메커니즘 (Mechanism)

1.  **가상 서버 구동**: 테스트 실행 시 로컬 환경에 실제 HTTP 서버(WireMock 등)를 띄웁니다.
2.  **스터빙 정의**: 특정 경로(`/api/v1/users/1`)로 요청이 올 때 어떤 응답(`200 OK`, `{"id": 1, "name": "test"}`)을 줄지 미리 설정합니다.
3.  **엔드포인트 전환**: 애플리케이션의 설정(`application-test.yml`)을 수정하여 실제 API 주소 대신 로컬 가상 서버 주소(`http://localhost:8080`)를 바라보게 합니다.

---

## 3. 왜 사용하는가? (Why?)

- **격리성**: 외부 서버의 장애나 데이터 상태에 영향을 받지 않는 독립적인 테스트가 가능합니다.
- **속도**: 실제 네트워크 통신보다 훨씬 빠른 속도로 응답을 받아볼 수 있습니다.
- **예외 상황 테스트**: 실제 서버에서는 발생시키기 어려운 500 에러, 타임아웃, 잘못된 데이터 형식 등을 자유롭게 시뮬레이션할 수 있습니다.
- **비용 절감**: 호출 시 비용이 발생하는 API(예: SMS 발송, 유료 API)를 비용 부담 없이 무제한으로 테스트할 수 있습니다.

---

## 4. 주요 도구 및 방식

- **WireMock**: 가장 널리 쓰이는 자바 기반 HTTP 모킹 라이브러리.
- **MockServer**: WireMock과 유사하게 가상 서버를 제공하는 도구.
- **Spring Cloud OpenFeign 테스트**: `FeignClient` 테스트 시 WireMock을 활용하여 실제 인터페이스 호출 흐름을 그대로 테스트 가능.
