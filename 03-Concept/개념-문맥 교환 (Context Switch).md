# 개념: 문맥 교환 (Context Switch)

## 📝 정의
CPU가 현재 실행 중인 프로세스나 스레드의 상태(Context)를 저장하고, 다음 차례의 프로세스/스레드 상태를 복구하여 실행을 전환하는 과정입니다.

## ⚙️ 작동 원리
1.  **상태 저장**: 현재 CPU 레지스터 값, 프로그램 카운터(PC) 등을 PCB(Process Control Block)나 TCB에 기록.
2.  **커널 모드 진입**: OS 스케줄러가 다음 실행 대상을 결정.
3.  **상태 복구**: 새로운 대상의 저장된 상태를 CPU 레지스터에 로드.
4.  **실행**: 이전에 멈췄던 지점부터 다시 실행.

## ⚠️ 치르는 비용 (Overhead)
- **CPU 점유**: 상태를 기록하고 읽는 동안 실제 비즈니스 로직은 멈춤.
- **캐시 오염 (Cache Miss)**: 새로운 프로세스가 들어오면 기존 CPU 캐시(L1, L2 등)에 쌓여있던 데이터가 무효화되어 성능이 일시적으로 저하됨.
