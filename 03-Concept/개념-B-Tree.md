## 한 문장 정의 (Definition)

> (사전적) 정렬된 상태를 유지하며 노드당 2개 이상의 자식을 가질 수 있는, 디스크 환경에 최적화된 자가 균형 탐색 트리(Self-Balancing Search Tree).
> (이해용) 수억 권의 책 사이에서 단 3~4번의 선반 이동만으로 원하는 책을 찾게 해주는 **'도서관 십진분류표'**.

## 해결하는 문제 (Problem Solved)

- **디스크 I/O 병목**: 이진 탐색 트리(BST)는 데이터가 많아질수록 트리의 높이가 깊어져 검색 시 디스크 Seek 횟수가 폭증하는 문제를 해결합니다.
- **검색 성능의 불균형**: 데이터 삽입/삭제 시에도 트리의 높이를 일정하게 유지하여, 최악의 경우에도 $O(\log N)$의 성능을 보장합니다.

## 치르는 비용 (Cost/Trade-off)

- **쓰기 성능 저하**: 노드가 가득 찼을 때 발생하는 노드 분할(Split) 및 재배치 연산으로 인해 데이터 삽입/수정 시 오버헤드가 발생합니다.
- **공간 낭비**: 각 노드는 항상 최소 50% 이상의 데이터만 채워지면 되므로, 완전하게 채워지지 않은 노드들로 인한 디스크 공간 사용량이 발생합니다.

## 동작 원리 (Mechanism)

### ASCII Diagram (M=3 인 경우의 구조)
```text
          [ 10 | 20 ]  <-- Root (최대 2개 키, 3개 자식)
         /     |     \
   [ 5 | 8 ] [13|17] [25|30] <-- Leaf (모든 리프는 동일 레벨)
```

### 핵심 메커니즘
1. **낮은 높이 (Low Height)**: 하나의 노드에 페이지 크기만큼 많은 키를 담아 '옆으로 넓은' 트리를 만듭니다. (일반적으로 3~4단계 내에 수억 개의 데이터 도달 가능)
2. **리프 노드의 균형**: 삽입 시 위에서 아래로 내려가는 것이 아니라, 리프 노드가 가득 차면 중간 값을 부모로 올리는 **Bottom-up** 방식으로 균형을 맞춥니다.
3. **디스크 블록 단위**: 노드 하나가 디스크의 한 블록(Page) 크기와 일치하도록 설계되어 한 번의 I/O로 최대한 많은 키를 메모리에 로드합니다.
