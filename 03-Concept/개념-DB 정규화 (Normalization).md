## 한 문장 정의

> (사전적) 함수적 종속을 기준으로 테이블을 분해하여 데이터 중복과 이상 현상을 제거하는 과정.
> (이해용) 한 사실은 한 곳에만 저장하도록 테이블을 쪼개는 작업.

## 해결하는 문제

- [[현상-DB 이상 현상]] — 삽입/갱신/삭제 이상 제거
- 데이터 중복으로 인한 불일치 방지

## 치르는 비용

- 테이블 수 증가 → JOIN 비용 증가
- 과도한 정규화는 조회 성능 저하로 이어질 수 있음
- 실무에서는 의도적 비정규화(반정규화)로 균형을 맞추기도 함

## 동작 원리

```
비정규 테이블:
┌──────┬──────┬──────┬──────┬──────┐
│ 학번 │ 이름 │ 과목 │ 성적 │ 학과 │
└──────┴──────┴──────┴──────┴──────┘
  → 이름/학과 중복, 이상 현상 발생
```

### 1NF — 원자값

모든 속성이 더 이상 쪼갤 수 없는 단일 값이어야 함

```
❌ { 수강과목: "수학, 영어, 물리" }  →  ✅ 행으로 분리
```

### 2NF — 부분 함수적 종속 제거

복합 기본키에서 키의 일부만으로 결정되는 속성을 분리

```
기본키: (학번, 과목코드)

학번 → 이름   ← 부분 종속 → 학생 테이블로 분리
(학번, 과목코드) → 성적   ← 완전 종속 → 유지
```

### 3NF — 이행적 함수적 종속 제거

비키 속성이 다른 비키 속성을 결정하는 관계를 분리

```
학번 → 학과코드 → 학과명
              ↑
         이 관계를 부서 테이블로 분리
```

### BCNF — 결정자가 반드시 후보키

3NF를 만족하더라도 결정자가 후보키가 아니면 위반

```
강사 → 과목   (강사는 후보키가 아님 → BCNF 위반)

분해:
강사 테이블: 강사 | 과목
수강 테이블: 학생 | 강사
```

### 정규화 단계 요약

```
1NF: 원자값
 ↓  부분 함수적 종속 제거
2NF: 완전 함수적 종속
 ↓  이행적 함수적 종속 제거
3NF: 비키 속성 간 종속 없음
 ↓  결정자 = 후보키 보장
BCNF: 모든 결정자가 후보키
```
