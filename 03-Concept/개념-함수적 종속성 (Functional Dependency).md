# 개념: 함수적 종속성 (Functional Dependency)

## 📝 정의
어떤 테이블에서 속성 A의 값을 알면 속성 B의 값을 유일하게 결정할 수 있을 때, "B는 A에 함수적으로 종속된다"고 하며 `A -> B`로 표기합니다.

## ⚙️ 구성 요소
- **결정자 (Determinant)**: 화살표 왼쪽의 A. 값을 결정하는 주체.
- **종속자 (Dependent)**: 화살표 오른쪽의 B. 결정자에 의해 결정되는 대상.

## 📐 종류
1.  **완전 함수적 종속**: 결정자가 복합키(A, B)일 때, (A, B) 전체에 의해서만 결정될 때.
2.  **부분 함수적 종속**: 결정자가 복합키인데, 키의 일부분(A 혹은 B)에 의해서도 종속될 때. (제2정규형의 제거 대상)
3.  **이행적 함수적 종속**: `A -> B`이고 `B -> C`일 때 `A -> C`가 성립하는 관계. (제3정규형의 제거 대상)

## ✅ 해결하는 문제
- 데이터를 어떤 기준으로 쪼개야(정규화) 정보의 손실 없이 중복을 제거할 수 있는지에 대한 **수학적 근거**를 제공합니다.
