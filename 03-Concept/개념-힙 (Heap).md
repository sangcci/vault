## 한 문장 정의

> (사전적) 부모 노드가 항상 자식 노드보다 크거나(Max-Heap) 작은(Min-Heap) 완전 이진 트리.
> (이해용) "항상 가장 우선순위가 높은 값이 루트에 있는 트리 — 최솟값·최댓값을 O(1)에 꺼내는 구조."

---

## 해결하는 문제

- 최솟값·최댓값 O(1) 조회, O(log n) 삽입·삭제
- Priority Queue 구현

---

## 치르는 비용

- **임의 탐색 O(n)**: 우선순위만 보장하고 정렬 순서는 없음 — 특정 값 탐색 불가
- BST처럼 중위 순회로 정렬된 결과를 얻을 수 없음

---

## 동작 원리

완전 이진 트리이므로 배열로 구현한다.

```
Min-Heap 예시:
        1
       / \
      3   5
     / \ / \
    4  8 6  9

배열: [1, 3, 5, 4, 8, 6, 9]
인덱스(1-based): 부모=i, 왼쪽=2i, 오른쪽=2i+1
```

**삽입 — sift-up:**
```
값 2 삽입:
[1, 3, 5, 4, 8, 6, 9, 2]  ← 맨 끝에 추가

2 < 부모(4) → swap
2 < 부모(3) → swap
2 > 부모(1) → 종료

결과: [1, 2, 5, 3, 8, 6, 9, 4]
```

**삭제(루트 추출) — sift-down:**
```
루트(1) 제거, 마지막 노드(4)를 루트로 이동
4 > 자식 중 최솟값(2) → swap
4 > 자식 중 최솟값(3) → swap
리프 도달 → 종료
```

| 연산 | 시간복잡도 |
|------|----------|
| peek (최솟값 조회) | O(1) |
| 삽입 | O(log n) |
| 삭제 | O(log n) |
