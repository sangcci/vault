## 한 문장 정의

> (사전적) 앞으로 가장 오랫동안 사용되지 않을 페이지를 교체하는 이론적 최적 알고리즘.
> (이해용) 미래를 아는 신이 하는 스케줄링 — 현실에선 구현 불가, 다른 알고리즘의 성능 상한선.

## 해결하는 문제

- 다른 페이지 교체 알고리즘의 성능을 평가하는 이론적 기준선 제공
- 최소 Page Fault 횟수 보장 (Belady's Anomaly 없음)

## 치르는 비용

- **미래 참조 패턴을 알 수 없음** → 실제 OS에서 구현 불가
- 시뮬레이션·분석 용도로만 사용

## 동작 원리

```
참조열: 1 2 3 4 1 2 5 1 2 3 4 5  (프레임 3개)

참조  프레임 상태        교체 선택 기준       Fault
 1    [1]                                     F
 2    [1,2]                                   F
 3    [1,2,3]                                 F
 4    [1,2,4] ← 3 교체   3: 다음 참조 10번째   F
                          1: 다음 참조  5번째
                          2: 다음 참조  6번째  → 3이 가장 나중
 1    [1,2,4]                                 HIT
 2    [1,2,4]                                 HIT
 5    [1,2,5] ← 4 교체   4: 다음 참조 11번째  F
                          1: 다음 참조  8번째
                          2: 다음 참조  9번째
 ...

총 Page Fault: 6회 (같은 입력에서 FIFO=9, LRU=8보다 적음)

[ 성능 비교 기준 ]
OPT ≤ LRU ≤ FIFO (Page Fault 횟수)
OPT는 모든 알고리즘의 하한(lower bound)
```

## 실제 사용처

- OS 알고리즘 성능 분석 도구 (오프라인 시뮬레이션)
- 캐시 알고리즘 설계 시 이론적 상한 계산
