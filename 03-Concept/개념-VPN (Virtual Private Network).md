## 한 문장 정의

> (사전적) 공개 네트워크(인터넷) 위에 암호화된 전용 터널을 생성해 사설망처럼 통신하는 기술.
> (이해용) **"인터넷이라는 공공도로 위에 아무도 못 보는 투명 파이프를 뚫는 것."**

## 해결하는 문제

- 인터넷 구간에서 패킷이 노출되는 스니핑 위협
- 외부에서 사내 Private IP 자원에 접근 불가 문제
- 실제 IP 노출로 인한 추적 위험

---

## 핵심 원리 — 목적지는 VPN 서버 IP만 안다

```
[내 PC (한국 IP)]
      │
      │ ── 암호화 터널 ──▶ [VPN 서버 (미국 IP)]
      │                           │
      │                           ▼
      │                   [목적지 서버]
      │                   "접속자 IP = 미국 VPN 서버"
      │                           ↑
      │               실제 내 IP는 모름
      │
      └── VPN 업체만 실제 내 IP를 앎
```

- 목적지 서버 입장에서 접속자는 **VPN 서버**
- 내 실제 IP는 VPN 업체 로그에만 존재
- VPN 업체가 로그를 보관하느냐 여부가 **익명성의 핵심**

---

## 악용이 되는 이유

같은 터널 메커니즘이 악용으로 이어진다:

```
합법적 사용                      악용
─────────────────────────────────────────
재택근무 → 사내망 접속          지역 제한 콘텐츠 우회
공공 와이파이 보호               불법 사이트 접근
해외 지사 연결                   해킹 후 IP 세탁
                                 다크웹 진입 경로
```

> 기술 자체는 중립 — **터널 출구 IP를 누가, 어디에 쓰느냐**가 차이

## SSL VPN — 재택근무 핵심 도구

### 동작 계층

```
OSI 7계층
──────────────────────────
7  응용   │ HTTP, SSH ...
6  표현   │ ← SSL VPN이 보호하는 범위
5  세션   │   (TLS가 이 구간 암호화)
4  전송   │
──────────────────────────
3  네트워크│ ← IPSec은 여기서 동작
2  데이터링크
1  물리
```

### 터널 개설 과정

```
[개발자 PC]                       [회사 VPN 서버]
     │                                  │
     │──── TLS 핸드셰이크 (443포트) ────▶│
     │◀─── 서버 인증서 전달 ─────────────│
     │──── CA 서명 검증 ────────────────▶│
     │◀─── 세션 키 교환 완료 ────────────│
     │                                  │
     │──── ID/PW, OTP 인증 ────────────▶│
     │◀─── 접근 가능 리소스 결정 ─────────│
     │                                  │
     └──────── 터널 개통 ────────────────┘
```

### 재택근무 시나리오

```
[개발자 집]
     │
     │ VPN 연결 (TLS 터널)
     ▼
[회사 VPN 서버]
     │
     ├──▶ GitLab (자체 호스팅)   ← Private IP
     ├──▶ Jira / Confluence      ← Private IP
     ├──▶ Jenkins CI/CD          ← Private IP
     ├──▶ 개발 DB (DBeaver)       ← Private IP
     └──▶ Grafana / Kibana       ← Private IP
```

단, SaaS는 이미 인터넷에 공개되어 있어 VPN 불필요:

```
VPN 필요                    VPN 불필요
──────────────────────────────────────
GitLab (자체 호스팅)        github.com
사내 Jira                   notion.so
사내 Jenkins                AWS 콘솔
사내 DB                     Google Workspace

기준: Private IP냐 Public IP냐
```

## 공공 와이파이에서 VPN

### VPN 없을 때 — 스니핑 위협

```
[내 PC]
   │ HTTP 평문 전송
   ▼
[카페 공유기] ◀── [공격자: 같은 와이파이]
   │                    │
   │          패킷 가로채기 (스니핑)
   ▼                    ▼
[인터넷]        id=hong&pw=1234 노출
```

### VPN 있을 때 — 스니핑 무력화

```
[내 PC]
   │ 암호화 터널
   ▼
[카페 공유기] ◀── [공격자: 스니핑 시도]
   │                    │
   │             ▒▒▒▒▒▒▒▒▒▒ (암호화된 쓰레기값)
   ▼
[VPN 서버] ── 복호화 ──▶ [목적지]
```

### HTTPS만으로 부족한 이유

```
HTTPS가 보호하는 것:          VPN이 추가로 보호하는 것:
─────────────────────────────────────────────────────
패킷 내용 (암호화 O)          DNS 쿼리
                              "google.com에 접속했다"는 사실
                              ↑ HTTPS여도 평문 노출

                              인증서 위조 MITM 공격
                              가짜 공유기 → 가짜 인증서 주입
                              ↑ VPN 터널 안에서는 불가
```

## SSL VPN vs IPSec 비교

```
SSL VPN                         IPSec
────────────────────────────────────────────────────
L4~L7 (TLS)                     L3 (IP 패킷 전체)
앱 단위 트래픽만 터널링          네트워크 전체 터널링
브라우저 / 경량 에이전트         전용 클라이언트 필수
TCP 443 (방화벽 통과 쉬움)       UDP 500, 4500
개인 원격접속                    본사↔지사 네트워크 연결
```
