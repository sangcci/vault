## 한 문장 정의

> (사전적) 홉 카운트를 메트릭으로 사용하는 거리 벡터 기반의 동적 라우팅 프로토콜.
> (이해용) 이웃 라우터에게만 "나는 여기까지 몇 홉이야"를 주기적으로 알려서 경로를 결정하는 방식.

---

## 해결하는 문제

- 소규모 네트워크에서 라우터 간 경로를 자동으로 공유

---

## 치르는 비용

- 최대 홉 수 15 제한 → 대규모 네트워크 불가
- 30초마다 전체 라우팅 테이블을 브로드캐스트 → 대역폭 낭비
- 느린 수렴 (Convergence) — 토폴로지 변경 시 전파가 느려 루프 발생 가능

---

## 동작 원리

**거리 벡터 방식 — 이웃에게만 전달**
```
라우터 A → 라우터 B에게: "나는 C까지 2홉이야"
라우터 B → 라우터 C에게: "나는 C까지 3홉이야"
→ 각 라우터는 이웃 정보만 보고 경로 결정
```

**알고리즘: 벨만-포드 (Bellman-Ford)**
```
목적지까지 비용 = 이웃까지 비용 + 이웃이 알려준 목적지까지 비용
→ 전체 토폴로지를 몰라도 됨
```

**벨만-포드 과정**
```
초기 상태: 자신까지 비용 = 0, 나머지 = ∞

네트워크: A -1- B -2- C -1- D
          A 입장에서 시작

라운드 1 (이웃에게 테이블 전달)
  A → B 에게 알림: "A까지 0홉"
  B 갱신: A=1, C=2, D=∞

라운드 2
  B → C 에게 전달
  C 갱신: A=3, B=2, D=1

라운드 3
  C → D 에게 전달
  D 갱신: A=4, B=3, C=1

→ 라우터 수 - 1 라운드 반복하면 수렴
→ 변경이 없을 때까지 30초마다 반복 브로드캐스트
```

**홉 카운트 15 제한**
```
홉 수 1~15  → 도달 가능
홉 수 16    → 무한대 (도달 불가능) 로 처리
→ 네트워크 규모가 커지면 사용 불가
```
