# Claude Code 컨텍스트 윈도우 절약 가이드

## 1. 컨텍스트 관리 전략

### 단일 목적 대화 유지 (ykdojo #14)
*   **핵심**: 하나의 대화(터미널 세션)에서는 하나의 명확한 목표에만 집중하세요.
*   **문제점**: 여러 주제를 한 대화에서 섞으면 컨텍스트가 오염되어 성능이 저하될 수 있습니다.
*   **해결책**: 새로운 터미널 탭을 열어 별도의 대화를 시작하세요.

### 선제적 압축 (ykdojo #8)
*   **핵심**: 대화가 길어져 컨텍스트가 가득 차면 성능이 저하될 때 사용합니다.
*   **워크플로우**:
    1.  `HANDOFF.md` 파일을 작성하도록 지시하여 이전 작업 내용을 요약합니다.
    2.  `/clear` 명령어로 새롭게 대화를 시작합니다.
    3.  `HANDOFF.md`에는 시도한 것, 성공한 것, 실패한 것, 다음 단계를 명확히 작성하여 다음 에이전트가 해당 파일만 읽고 작업을 이어갈 수 있도록 합니다.

### /context로 X-Ray 비전 (Ado #15)
*   **핵심**: `/context` 명령어를 사용하여 무엇이 컨텍스트를 차지하는지 정확히 확인합니다.
*   **확인 가능한 요소**: 시스템 프롬프트 크기, MCP 서버 프롬프트, 메모리 파일, 로드된 스킬과 에이전트, 대화 기록.
*   **최적화**: 불필요한 요소를 비활성화하거나 언로드하여 컨텍스트 사용량을 줄입니다.
*   **조언**: 일반적으로 10개 미만의 MCP와 80개 미만의 활성 도구를 유지하는 것이 최적입니다.

### 자동 압축 비활성화 (ykdojo #8)
*   **핵심**: Claude Code의 자동 컨텍스트 압축 기능을 비활성화하고 수동으로 관리합니다.
*   **이유**:
    *   자동 압축은 45k 토큰을 예약하여 실제 사용 가능한 컨텍스트를 줄입니다.
    *   압축 시점을 직접 제어하여 중요한 정보 손실을 방지할 수 있습니다.
    *   `HANDOFF.md` 방식이 더 명확하고 검증 가능합니다.

## 2. 시스템 프롬프트 슬림화 (ykdojo Tip #15)

### 시스템 프롬프트를 줄여야 하는 이유
*   더 많은 코드 파일과 대화 기록을 컨텍스트에 담을 수 있습니다.
*   응답 속도 향상 (처리할 토큰이 적음).
*   비용 절감 (토큰 사용량 감소).

### 슬림화 방법
*   장황한 예시: 도구 설명에 포함된 불필요하게 긴 예시 제거.
*   반복적인 지시: 여러 곳에서 반복되는 동일한 지시사항 제거.
*   과도한 안전 경고: 너무 자주 반복되는 경고 메시지 제거.
*   사용하지 않는 도구 설명: 거의 사용되지 않는 도구의 긴 설명 제거.

**주의**: 시스템 프롬프트 슬림화는 전문가의 영역이며, 잘못 수정하면 Claude의 성능이 심각하게 저하될 수 있습니다. 충분한 이해 없이 시도하지 마세요.
