# 요약: 개발자 상기님의 AI 활용 아침 루틴과 생산성 워크플로우

해당 영상은 개발자 '상기(sangcci)'님의 **아침 루틴과 AI를 활용한 생산성 워크플로우**를 소개하고 있습니다.

### 1. 정보 수집 및 AI 자동화 (Ingestion)
사용자가 매일 접하는 방대한 정보를 AI(Claude)를 통해 효율적으로 필터링하고 요약하여 Obsidian으로 가져오는 과정이 핵심입니다.

*   **RSS & 뉴스레터:** Feedley와 뉴스레터를 훑어보며 관심 있는 글을 선별합니다. 긴 영문 아티클은 브라우저 AI나 Claude 스킬을 이용해 즉시 요약합니다.
*   **유튜브 & 아티클 요약:** 직접 만든 'Claude 스킬'을 사용하여 유튜브 URL이나 웹 페이지 URL을 주면, AI가 자막/본문을 긁어와 **Markdown 파일로 요약본을 생성**해 Obsidian 볼트에 저장합니다.
*   **회의/대화 기록:** 아침 커피 타임 대화를 녹음 후 텍스트로 전사(Transcription)하고, 이를 Claude가 정리하여 팀 내부 레포지토리에 기록합니다.

### 2. Obsidian 기반 지식 관리 (Knowledge Management)
수집된 파편화된 정보를 체계화하는 방식입니다.

*   **데일리 로그 자동화:** 그날 작업한 코드 변경 사항, 클로드와 나눈 대화, 일정 관리 도구(Things)의 완료 내역 등을 모아 **데일리 로그를 자동 생성**합니다.
*   **상향식 정리:** 데일리 로그 -> 위클리 -> 먼슬리 -> 이어리(Yearly)로 이어지는 상향식(Bottom-up) 정리 구조를 지향합니다.
*   **주석 및 개인화:** AI가 요약해준 문서에 본인이 직접 이탤릭체나 하이라이트를 추가하며 **자신만의 언어로 다시 정리(Annotation)**합니다.

### 3. 생산성 도구 및 환경 (Tools)
최대한 마우스를 쓰지 않고 키보드 중심으로 흐름을 유지하는 도구들을 사용합니다.

*   **터미널 환경:** Ghostty 터미널 + tmux + Neovim 기반의 CLI 환경을 선호하며, Claude Code를 최신으로 유지합니다.
*   **키보드 중심 웹 서핑:** `Vimium`(크롬 익스텐션)을 사용하여 마우스 없이 웹을 탐색하고, `Homerow`나 `skhd`를 이용해 앱 간 이동을 자동화합니다.
*   **하드웨어:** 스플릿 키보드를 사용하여 신체적 부담을 줄이고 타이핑 효율을 높입니다.

### 4. 학습 철학: 밸런스 유지
*   **트렌드 vs 본질:** 아침에는 SNS나 뉴스레터로 최신 트렌드를 파악하고, 저녁에는 O'Reilly 같은 전자책으로 깊이 있는 지식을 습득하며 균형을 맞춥니다.
*   **기록의 중요성:** 모든 활동을 Markdown으로 남겨 검색 가능하게 만들고, 나중에 책을 쓰거나 발표할 수 있는 자산으로 쌓아둡니다.
