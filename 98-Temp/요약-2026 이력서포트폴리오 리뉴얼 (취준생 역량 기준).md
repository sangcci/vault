# 2026 이력서·포트폴리오 가이드 요약 (취준생 역량 기준)

---

## 1. 취준생 역량 기준: 60% 기본기 + 40% 강점

### 기본기 (60%) — 반드시 채워야 할 것
> 면접관은 "기본기를 모르는 거 아니야?"라는 선입견을 먼저 갖고 있음.
> 이력서에서 먼저 해소해야 함.

| 역량 | 설명 |
|---|---|
| EXPLAIN 분석 + 쿼리 개선 | MySQL Workbench에서 실행 계획 분석 후 개선 |
| 로컬 캐시 | API 조회 응답 속도 개선 |
| Redis 캐시 | 성능 개선 + 데이터 정합성 보완 |
| 동기/비동기 구분 | 비동기 처리를 통한 성능 개선 |

### 강점 (40%) — 차별화 요소
> 어느 정도 연차가 있으면 이것도 "기본기"가 될 수 있음.
> 취준생 기준으로는 희소성 높음.

| 역량 | 설명 |
|---|---|
| JVM 메모리/GC 튜닝 | 힙 영역 비율 및 GC 파라미터 조정, 안정성 개선 |
| tcpdump 네트워크 분석 | 패킷 분석으로 장애 원인 추적 및 재발 방지 |
| Kafka 이벤트 아키텍처 설계 | 토픽/컨슈머 구조 및 처리 전략 수립 |
| Netty 기반 SSE | 실시간 이벤트 전송 구조 구성, 안정성 확보 |

## 2. 이력서 작성 기준

### 이력서의 본질
> 면접관은 하루 10~20개 이력서를 검토. "읽기 좋은 문서"가 목표.

### 각 항목 방향성

| 항목 | 핵심 방향 |
|---|---|
| **제목** | "이름 이력서"로만. 감성 문구 금지 |
| **사진** | 가능하면 필수. SI → 증명사진, 스타트업 → 자연스러운 사진 |
| **지원 동기** | "내가 이 팀에 어떤 도움을 줄 수 있는가" 2~3줄. JD 기반으로 |
| **자기소개** | 팩트 기반. 열정/꼼꼼함 어필 지양. 수치/경험으로 |
| **프로필** | 이름, 이메일, 경력, GitHub, 블로그, 포트폴리오 링크. 주소 제외 |
| **자격증** | SI/금융권은 정보처리기사 필수. 스타트업/서비스 회사는 불필요 |

### 프로젝트 작성 기준
- **개수**: 3~4개 (이력서 2장 내외)
- **기술 스택**: 버전 명시 (Java X → Java 17). Git/Slack/Notion 제외
- **참여 인력**: "10인" X → "FE 2명 / BE 3명" O
- **서비스 개요**: 1줄로 압축

### 프로젝트 내용 구조 (핵심)
```
문제 + 해결 + 결과 + 도메인
```
- 최상단에 문제 해결 역량 3~4줄 (가장 잘한 것부터)
- 이후 단순 구현 2~3줄 ("어떤 기술로 무엇을 구현했고, 어떤 도움이 됐는가")
- 마지막에 맞닿은 영역 역량 (전시 파트 → React/Vue, 플랫폼 파트 → k8s/모니터링)

**작성 예시**
```
❌ "Redis 캐시 적용"
✓  "질문 게시글 리스트 조회 시 느린 응답 발생 및 Redis 캐시 적용으로 평균 응답 속도 350ms → 40ms 개선"
```

## 3. 포트폴리오 작성 기준

### 이력서 vs 포트폴리오 차이
| | 이력서 | 포트폴리오 |
|---|---|---|
| 역할 | 전체 압축 요약 | 핵심 문제 해결 상세 풀어쓰기 |
| 분량 | 한 줄로 압축 | 문제/해결/결과 각 3줄 = 총 9줄 |
| 개수 | 프로젝트 3~4개 전부 | 가장 자신 있는 4~5개 선별 |

### 포트폴리오 구조 (한 덩어리)
```
제목 (이력서의 문제 해결 한 줄 그대로)
  ↓
그림 (시퀀스 다이어그램 / 아키텍처 변경 전·후 / 테스트 결과)
  ↓
문제 (3줄)
해결 (3줄, 앱 레벨 구현 내용 포함 가능)
결과 (3줄, 수치 필수)
```

### 그림 관련
- 백엔드는 앱 화면 캡처 금지 → **구조와 흐름** 표현
- 추천 툴: `draw.io`, `excalidraw.com`
- 아이콘 대신 텍스트 사각형으로 (가독성 우선)
- 코드 캡처 금지 → 흐름/수치/테스트 결과로 증명

### 문제 해결 경험이 없을 때
> "1000TPS가 들어오면 어디가 먼저 터질까?" 가정하고 개선 포인트 리스트업.
> 기존 프로젝트를 다시 돌려보고 개선 후 이력서에 추가.

## 4. 자기소개서 전략

### 구조
```
1,000자 기준:
[소제목 50자 + 내용 450자] × 2 문단
```
- 각 문단 = JD의 요구사항 1개와 매핑
- 소제목만 읽어도 핵심 파악 가능하게

### 합격률을 높이는 방법
```
JD의 지원 자격/우대 사항 → 내 프로젝트와 매핑 → 자기소개서 문단 배정
```

---

## 5. 차별화 전략 (나만의 무기)

### 블로그 활용
- Redis, MySQL 등 범용 기술 주제로 **딥다이브 글 1개** 작성
- 이력서 최상단에 눈길 가는 링크 문구로 배치
- **약점 커버 용도**로 활용 (SI 출신 → MySQL 딥다이브)

### 자기소개 섹션
- "첫 질문이 나오도록" 설계 (궁금증 유발)
- 면접에서 아키텍처를 직접 그리며 설명 → 강한 임팩트
