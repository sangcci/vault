# 1과목 소프트웨어 설계

3번 문제: 자료 흐름도(DFD) 표기법
문제: 자료 흐름도(DFD)의 각 요소별 표기 형태의 연결이 옳지 않은 것은?
① 자료 흐름(Data Flow) : 화살표
② 처리(Process) : 원
③ 자료 저장소(Data Store) : 직선(평행선)
④ 단말(Terminator) : 오각형
 * 정답: ④
 * 해설: 자료 흐름도에서 단말(Terminator) 또는 외부 엔티티(External Entity)는 오각형이 아닌 사각형으로 표기합니다. 그 안에 이름을 기입하는 방식입니다.
8번 문제: 사용자 인터페이스(UI)의 종류
문제: UI의 종류로 멀티 터치(Multi-touch), 동작 인식(Gesture Recognition) 등 사용자의 자연스러운 움직임을 인식하여 서로 주고받는 정보를 제공하는 사용자 인터페이스를 의미하는 것은?
① GUI(Graphical User Interface)
② OUI(Organic User Interface)
③ NUI(Natural User Interface)
④ CLI(Command Line Interface)
 * 정답: ③
 * 해설: **NUI(Natural User Interface)**는 키보드나 마우스 같은 장치 대신 사람의 자연스러운 신체 움직임(음성, 지문, 터치, 동작 등)을 이용해 기기를 제어하는 인터페이스를 말합니다.
11번 문제: 객체지향 구성 요소
문제: 객체지향의 주요 구성 요소 중 데이터와 데이터를 처리하는 메소드를 묶어 놓은 하나의 소프트웨어 모듈을 무엇이라고 하는가?
① 클래스(Class)
② 객체(Object)
③ 상속(Inheritance)
④ 관계(Relationship)
 * 정답: ②
 * 해설: 데이터(속성)와 그 데이터를 처리하는 함수(메소드)를 하나로 묶어 놓은 실체를 **객체(Object)**라고 합니다.
   * 클래스(Class): 공통된 속성과 연산을 갖는 객체들의 집합(틀).
   * 상속(Inheritance): 상위 클래스의 속성과 연산을 하위 클래스가 물려받는 것.
14번 문제: 결합도(Coupling)
문제: 결합도(Coupling)에 대한 설명으로 틀린 것은?
① 데이터 결합도(Data Coupling)는 두 모듈이 매개변수로 자료를 전달할 때 자료 구조 형태로 전달되어 이용될 때 데이터가 결합되어 있다고 한다.
② 내용 결합도(Content Coupling)는 하나의 모듈이 직접적으로 다른 모듈의 내용을 참조할 때 두 모듈은 내용적으로 결합되어 있다고 한다.
③ 공통 결합도(Common Coupling)는 두 모듈이 동일한 전역 데이터를 접근한다면 공통 결합되어 있다고 한다.
④ 결합도(Coupling)는 두 모듈 간의 상호작용, 또는 의존도 정도를 나타내는 것이다.
 * 정답: ①
 * 해설: 1번 설명은 **스탬프 결합도(Stamp Coupling)**에 대한 설명입니다.
   * 데이터 결합도(Data Coupling): 모듈 간의 인터페이스가 단순한 **자료 요소(파라미터)**로만 구성될 때의 결합도로, 가장 낮은(좋은) 결합도입니다.
18번 문제: 미들웨어(Middleware)
문제: 미들웨어에 대한 설명으로 옳지 않은 것은?
① DB는 데이터베이스 벤더에서 제공하는 클라이언트에서 원격의 데이터베이스와 연결하기 위한 미들웨어이다.
② WAS는 사용자의 요구에 따라 변하는 동적인 콘텐츠를 처리하기 위해 사용되는 미들웨어이다.
③ MOM은 메시지 기반의 비동기형 메시지를 전달하는 방식의 미들웨어이다.
④ RPC는 코바(CORBA) 표준 스펙을 구현한 객체 지향 미들웨어이다.
 * 정답: ④
 * 해설: 4번 설명은 **ORB(Object Request Broker)**에 대한 설명입니다.
   * RPC(Remote Procedure Call): 응용 프로그램의 프로시저를 마치 로컬 프로시저처럼 호출하는 방식의 미들웨어입니다.

# 2과목 - 소프트웨어 개발

22번 문제: 코드 검사 시 발견되는 오류의 종류
문제: 코드 검사 수행 시 발견된 오류와 그 설명으로 틀린 것은?
① 데이터 오류(DA; Data Error)는 데이터 유형 정의, 변수 선언, 매개 변수 등에서 나타나는 오류이다.
② 기능 오류(FN; Function Error)는 서브루틴이나 블록이 수행하는 방법(How)이 잘못되어 있는 오류이다.
③ 성능 오류(PF; Performance Error)는 프로그램을 수행하며 요구되는 성능을 만족시키지 못하는 오류이다.
④ 문서 오류(DC; Documentation Error)는 프로그램 구성 요소인 선언 부분, 잘못되거나 불필요한 주석 등을 의미한다.
 * 정답: ②
 * 해설: **기능 오류(Function Error)**는 서브루틴이나 블록이 잘못된 것(What)을 수행하는 오류를 말합니다. 선택지에서 설명하는 '수행하는 방법(How)이 잘못된 것'은 **논리 오류(LO; Logic Error)**에 해당합니다.
26번 문제: 역공학(Reverse Engineering)
문제: 다음 설명에 해당하는 것은?
> 기존 소프트웨어를 분석하여 소프트웨어 개발 과정과 데이터 처리 과정을 설명하는 분석 및 설계 정보를 재발견하거나 다시 만들어 내는 활동이다.
> 
① Analysis
② Restructuring
③ Reverse Engineering
④ Migration
 * 정답: ③
 * 해설: 일반적인 개발 단계와는 반대로(Reverse) 기존 소프트웨어를 분석하여 새로운 정보를 재발견하는 활동을 **역공학(Reverse Engineering)**이라고 합니다.
   * 재구성(Restructuring): 외적인 동작은 바꾸지 않고 코드의 구조만 향상시키는 활동
   * 이식(Migration): 기존 소프트웨어를 다른 운영체제나 하드웨어 환경에서 사용할 수 있도록 변환하는 활동

문제: ISO/IEC 25010의 소프트웨어 품질 특성 중 사용성(Usability)에 해당하지 않는 것은?

30번 문제: 자료 구조 - 데크(Deque)
문제: 다음 설명이 의미하는 것은?
>  * 삽입과 삭제가 리스트의 양쪽 끝에서 발생할 수 있는 형태이다.
>  * 입력이 한쪽에서만 발생하고 출력은 양쪽에서 일어날 수 있는 입력 제한과, 입력은 양쪽에서 일어나고 출력은 한 곳에서만 이루어지는 출력 제한이 있다.
> 
① 스택
② 큐
③ 다중 스택
④ 데크
 * 정답: ④
 * 해설: 삽입과 삭제가 양쪽 끝에서 모두 가능한 자료 구조는 **데크(Deque, Double Ended Queue)**입니다.
   * 스택(Stack): 한쪽 끝으로만 삽입/삭제가 일어나는 LIFO 구조
   * 큐(Queue): 한쪽에서는 삽입, 다른 한쪽에서는 삭제가 일어나는 FIFO 구조

문제: 검증 검사 기법 중 개발자의 장소에서 사용자가 개발자 앞에서 행하는 기법이며, 일반적으로 통제된 환경에서 사용자와 개발자가 함께 확인하면서 수행되는 검사는?
① 동치 분할 검사
② 형상 검사
③ 알파 검사
④ 베타 검사

35번 문제: 이진 검색(Binary Search) 비교 횟수
문제: 다음과 같이 레코드가 구성되어 있을 때, 이진 검색 방법으로 'F'를 찾을 경우 비교되는 횟수는?
> A B C D E F G H I J K L M N
> 
① 4
② 5
③ 6
④ 7
 * 정답: ①
 * 해설: 데이터 총 개수를 14개(1~14)로 가정했을 때의 풀이 과정입니다.
   * 1회 비교: 중간값 M = (1+14) / 2 = 7.5(정수부 7). 7번째 값인 'G'와 비교. 'F'는 6번째이므로 왼쪽(1~6)을 탐색합니다.
   * 2회 비교: 중간값 M = (1+6) / 2 = 3.5(정수부 3). 3번째 값인 'C'와 비교. 'F'는 6번째이므로 오른쪽(4~6)을 탐색합니다.
   * 3회 비교: 중간값 M = (4+6) / 2 = 5. 5번째 값인 'E'와 비교. 'F'는 6번째이므로 오른쪽(6)을 탐색합니다.
   * 4회 비교: 중간값 M = (6+6) / 2 = 6. 6번째 값인 'F'를 발견합니다.
     따라서 총 4회의 비교가 필요합니다.

# 3과목 - 데이터베이스 구축

48번 문제: 로킹(Locking) 기법 - 2단계 로킹 규약
문제: 로킹 기법에서 2단계 로킹 규약에 대한 설명으로 옳은 것은?
① 트랜잭션은 Lock만 수행할 수 있고, Unlock은 수행할 수 없는 확장 단계가 있다.
② 트랜잭션이 Unlock과 Lock을 동시에 수행할 수 있는 단계를 병렬 전환 단계라 한다.
③ 한 트랜잭션이 Unlock 후 다른 데이터 아이템을 Lock 할 수 있다.
④ 교착상태를 일으키지 않는다.
 * 정답: ①
 * 해설: **2단계 로킹 규약(2PL)**은 트랜잭션 내의 모든 로킹 연산이 첫 번째 언록 연산 이전에 수행되어야 하는 규약입니다.
   * 확장 단계: 새로운 Lock은 가능하지만 Unlock은 할 수 없는 단계입니다.
   * 축소 단계: 새로운 Unlock은 가능하지만 Lock은 할 수 없는 단계입니다.
   * 주의: 직렬성을 보장하지만 **교착상태(Deadlock)**가 발생할 수 있다는 단점이 있습니다.
58번 문제: 분산 데이터베이스의 특징
문제: 분산 데이터베이스의 특징에 대한 설명으로 틀린 것은?
① 지역 서버의 고유 데이터에 대한 작업은 중앙 서버의 통제 없이 자유롭게 수행할 수 있다.
② 새로운 지역 서버를 추가하거나 장비를 추가하는 등의 작업이 용이하다.
③ 위치 투명성, 중복 투명성, 병행 투명성, 장애 투명성을 목표로 한다.
④ 데이터베이스 설계 및 소프트웨어 개발이 쉽고, 전반적인 시스템의 성능이 향상된다.
 * 정답: ④
 * 해설: 분산 데이터베이스는 여러 곳에 데이터가 흩어져 있으므로 관리 포인트가 많아져 설계 및 소프트웨어 개발이 오히려 어렵고 복잡합니다.

69번 문제: FIFO 페이지 교체 알고리즘 계산
문제: 4개의 페이지를 수용할 수 있는 주기억장치가 있으며, 초기에는 모두 비어 있다고 가정한다. 다음의 순서로 페이지 참조가 발생할 때, FIFO 페이지 교체 알고리즘을 사용할 경우 페이지 결함의 발생 횟수는?
> 페이지 참조 순서 : 0, 1, 2, 3, 0, 1, 4, 0, 1, 2, 3, 4
> 
① 7회 ② 8회 ③ 9회 ④ 10회
 * 정답: ④ (10회)
 * 해설: **FIFO(First-In, First-Out)**는 가장 먼저 들어온 페이지를 먼저 교체하는 방식입니다.
   * 0, 1, 2, 3 적재 (결함 4회 발생)
   * 0, 1 참조 (이미 있음, 결함 없음)
   * 4 참조: 가장 오래된 0을 제거하고 4 적재 (결함 5회)
   * 0 참조: 그 다음 오래된 1을 제거하고 0 적재 (결함 6회)
   * 1 참조: 그 다음 오래된 2를 제거하고 1 적재 (결함 7회)
   * 2 참조: 그 다음 오래된 3을 제거하고 2 적재 (결함 8회)
   * 3 참조: 그 다음 오래된 4를 제거하고 3 적재 (결함 9회)
   * 4 참조: 그 다음 오래된 0을 제거하고 4 적재 (결함 10회)
72번 문제: IPv6 주소 체계

③ IPv4는 호스트 주소를 자동으로 설정하며 유니캐스트(Unicast)를 지원한다.
④ IPv4는 클래스별로 네트워크와 호스트 주소의 길이가 다르다.
 * 정답: ①
 * 해설: IPv6의 패킷 헤더는 32 octet이 아니라 40 octet의 고정된 길이를 가집니다.

# 5과목 - 정보시스템 구축 관리

93번 문제: 암호화 알고리즘의 분류
문제: 블록 암호화 방식과 해시 암호화 방식을 나열한 것이다. 다음 중 유형이 다른 하나는?
① DES ② SNEFRU ③ MD5 ④ SHA
 * 정답: ①
 * 해설: DES는 개인키를 이용하는 블록 암호화 알고리즘입니다. 나머지는 모두 **해시 암호화 방식(Hash Function)**입니다.
   * SNEFRU: 1990년 발표된 해시 함수
   * MD5: MD4를 대체하기 위해 고안된 해시 함수
   * SHA: 미국 국가안보국(NSA)이 설계한 해시 함수 시리즈
95번 문제: 서비스 거부 공격(DoS) - SYN Flooding
문제: 공격자가 가상의 클라이언트로 위장하여 3-way-handshake 과정을 의도적으로 중단시킴으로써 공격 대상지인 서버가 대기 상태에 놓여 정상적인 서비스를 수행하지 못하게 하는 공격 방법은?
① SYN Flooding ② SMURFING ③ Land ④ TearDrop
 * 정답: ①

   * TearDrop: 프래그먼트 오프셋(Fragment Offset) 값을 조작하여 재조립 시 오류 유발
96번 문제: 정보보안 3요소 (가용성)
​문제: 정보 보안의 3요소 중 하나로, 인가된 사용자가 필요로 할 때 언제든지 정보 자산에 접근하여 사용할 수 있음을 의미하는 것은?
① 기밀성(Confidentiality)
② 무결성(Integrity)
③ 가용성(Availability)
④ 부인 방지(Non-Repudiation)
​정답: ③
​해설: **가용성(Availability)**은 권한이 있는 사용자가 원할 때 언제든 서비스를 이용할 수 있는 능력을 말합니다.
​99번 문제: 로그 파일 (wtmp)
​문제: 리눅스/유닉스 시스템에서 사용자의 성공적인 로그인/로그아웃 정보와 시스템의 시작/종료 시간 정보를 저장하는 로그 파일은?
① wtmp
② utmp
③ lastlog
④ btmp
​정답: ①
​해설: wtmp는 모든 사용자의 접속 이력(로그인/로그아웃)과 시스템 리부팅 정보를 기록하며 last 명령어로 확인 가능합니다.
100번 문제: 프로젝트 관리 3P (People, Problem, Process)
문제: 소프트웨어 프로젝트 관리를 효율적으로 수행하기 위한 3P 중 소프트웨어 프로젝트를 수행하기 위한 Task Framework의 고려와 가장 연관되는 것은?
① People ② Problem ③ Product ④ Process
 * 정답: ④
 * 해설: 프로젝트 관리의 3대 요소(3P) 중 **프로세스(Process)**는 소프트웨어 개발에 필요한 전체적인 작업 계획 및 구조(Framework)를 의미합니다.
